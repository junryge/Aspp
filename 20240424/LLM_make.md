## 1.1 왜 "개인형 LLM"인가?

상용 LLM(ChatGPT, Claude 등)은 강력하지만, 연구·산업 현장에서는 다음과 같은 한계가 있다.

| 구분 | 상용 클라우드 LLM | 개인형 LLM (이 가이드의 목표) |
|------|-------------------|--------------------------------|
| 데이터 보안 | 외부 서버로 전송 | **로컬 처리, 외부 유출 없음** |
| 도메인 적합성 | 일반 상식 위주 | **내 분야(과학·반도체·물류 등)에 특화** |
| 비용 | 호출당 과금 | **고정비(하드웨어) + 무료 추론** |
| 커스터마이징 | 제한적 | **스킬·프롬프트·모델 자유 변경** |
| 오프라인 사용 | 불가 | **인터넷 없이도 동작** |

이 프로젝트의 핵심 컨셉은 다음 세 가지다.

1. **로컬 우선(Local-First)** — 모델·데이터·로그가 모두 내 PC/서버에 머문다.
2. **스킬 기반(Skill-Based)** — 분야별 지식을 "스킬(skill)" 단위로 모듈화해 LLM에 주입한다.
3. **하이브리드(Hybrid)** — 필요시 외부 API(Claude, OpenAI 등)와 로컬 GGUF 모델을 함께 사용한다.

## 1.2 우리가 만들 것 — Scientific-Assistant 전체 구조

```
┌─────────────────────────────────────────────────────────┐
│              Scientific-Assistant (웹 UI)                │
│   http://localhost:10009  ←  Flask 기반 채팅 인터페이스   │
└─────────────────────────────────────────────────────────┘
            │                          │
            ▼                          ▼
┌────────────────────┐       ┌────────────────────────┐
│   Skills Engine    │       │   Model Backend        │
│  355+ 과학·개발 스킬 │       │  ① Claude API (원격)   │
│  (자동 로딩/검색)    │       │  ② GGUF 로컬 모델       │
└────────────────────┘       │  ③ Ollama / llama.cpp   │
            │                 └────────────────────────┘
            ▼                          │
┌─────────────────────────────────────────────────────────┐
│   Data / Tools Layer                                     │
│   - Logpresso (로그 분석)                                │
│   - Harness Bridge (외부 도구 연결)                       │
│   - users.json / api_config.json (인증·설정)             │
└─────────────────────────────────────────────────────────┘
```

### 구성 요소 한눈에 보기

- **`app.py`** — 진입점(엔트리포인트). Flask 앱을 띄운다.
- **`demos_v1/`** — 라우팅·모델·스킬 처리를 담는 핵심 패키지.
- **`scientific-skills/`** — 355개 이상의 도메인 스킬(과학 174, 개발도구 52, 에이전트 117, 가이드 12).
- **`harness_bridge.py`** — 외부 하니스(harness)·MCP 도구 연결 어댑터.
- **`logpresso_client.py`** — Logpresso 로그 분석 시스템 연동.
- **`TOKEN.TXT`** — Claude/Anthropic API 키(한 줄).
- **`api_config.json`** — 모델·엔드포인트 설정.

| 사용 시나리오 | 최소 사양 | 권장 사양 |
|---------------|-----------|-----------|

## 1.9 자주 묻는 질문 (FAQ)

**Q1. 인터넷 없이도 쓸 수 있나요?**
A. 5장에서 다룰 **GGUF 로컬 모델**을 사용하면 완전 오프라인으로 동작한다. 단, 처음 모델 파일을 받을 때만 네트워크가 필요하다.

**Q2. GPU가 없는데 가능한가요?**
A. 가능하다. ① Claude API 모드, 또는 ② CPU 추론용 양자화 GGUF(예: Q4_K_M) 모델을 쓰면 된다. 다만 응답이 느려진다.

**Q3. 회사 보안 정책상 외부 API 호출이 안 됩니다.**
A. 처음부터 **5장의 로컬 모델 모드**로 진행해도 된다. 1~4장의 일부 예제만 건너뛰면 된다.

**Q4. 데이터(예: 사내 PDF)를 학습시키려면?**
A. 두 가지 길이 있다. ① **RAG**(6장): 학습 없이 검색으로 주입. ② **파인튜닝**(9장): 모델 가중치 자체를 미세조정. 대부분의 경우 **RAG부터** 시작하는 것이 비용·효율 면에서 유리하다.
