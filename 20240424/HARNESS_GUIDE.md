# ASAS 하네스(Harness) 시스템 안내

## 한 줄 요약

> **하네스 = "질문을 받아 알아서 적절한 AI 모델 + 관련 도메인 지식 + 보조 도구를 골라 답변하는 지휘자"**

사용자는 그냥 **자연어로 질문**만 하면, 하네스가 뒤에서 ① 어떤 모델이 답변에 가장 적합한지, ② 어떤 스킬을 함께 써야 하는지, ③ 여러 단계를 어떻게 병렬로 처리할지를 자동 결정합니다.

---

## 1. 왜 필요한가 (고객 입장)

기존 챗봇의 한계:
- ❌ 모델 한 종류만 사용 → 코드는 잘 짜지만 보고서는 부실 (또는 그 반대)
- ❌ 사용자가 매번 어떤 도구·문서·예시를 첨부할지 직접 결정
- ❌ 모델 한 곳이 다운되면 답변 불가
- ❌ 복잡한 작업은 한 번에 끝나지 않고 사용자가 단계마다 지시 필요

ASAS 하네스가 해결:
- ✅ **자동 모델 선택**: 7개 LLM 중 질문 성격에 맞는 모델 선택 (이미지·코드·보고서·간단 Q&A 별)
- ✅ **자동 스킬 매칭**: 397개 도메인 스킬 중 관련된 것만 자동 로드
- ✅ **자동 폴백**: 모델 1개 다운되어도 다음 모델로 자동 전환 (사용자 모름)
- ✅ **병렬 처리**: 여러 스킬을 동시 실행해 답변 속도 단축
- ✅ **사용자 학습 곡선 없음**: 그냥 한국어로 물어보면 끝

---

## 2. 전체 흐름 (사용자 → 답변까지)

```
┌─────────────────────────────────────────────────────────────┐
│ 사용자 입력                                                   │
│ "주간보고 PPT 만들어줘"                                       │
└─────────────────┬────────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────────────────┐
│ ① 라우터 (router.py)                                          │
│ - 입력 분석: 복잡도, 키워드, 첨부파일                          │
│ - 결정: GLM-5 모델 (보고서 생성 키워드 감지)                   │
└─────────────────┬────────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────────────────┐
│ ② 스킬 자동 선택 (skills.py)                                  │
│ - 트리거 단어로 397개 스킬 중 관련 N개 추출                    │
│ - 예: weekly-report, pptx, scientific-writing                │
└─────────────────┬────────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────────────────┐
│ ③ 도메인 지식 검색 (knowledge.py, BM25)                       │
│ - 사용자 자체 등록 문서 60개+ 에서 관련 자료 자동 첨부          │
│ - 예: "지난주 보고서.md" 자동 매칭 → 컨텍스트 주입              │
└─────────────────┬────────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────────────────┐
│ ④ 멀티에이전트 병렬 실행 (engine.py)                          │
│ - 스킬을 그룹으로 나눠 동시 호출                               │
│ - 각 그룹별 적정 모델 자동 할당                                │
└─────────────────┬────────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────────────────┐
│ ⑤ 응답 합성 + 산출물 생성                                     │
│ - PPT 자동 생성 → 다운로드 링크                                │
│ - 답변 텍스트 + 출처 표시                                      │
└─────────────────┬────────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────────────────────────┐
│ 사용자 화면                                                   │
│ "20260429_주간보고.pptx 다운로드"                             │
│ + 답변 본문 + 사용된 스킬 목록 + 자동 선택된 모델 표시          │
└─────────────────────────────────────────────────────────────┘
```

전체 5단계가 **3-15초 내** 완료되며, 사용자는 입력 한 번만 하면 됩니다.

## 3. 핵심 컴포넌트

### 3.1 모델 라우터 (자동 모델 선택)

7개의 LLM 중 자동 선택:

| 모델 | 특기 | 선택되는 경우 |
|---|---|---|
| **Qwen3-VL-30B** | 이미지 분석 | 이미지 첨부 시 |
| **GLM-5** | 종합 분석·보고서 | 복잡한 분석, PPT, 긴 보고서 |
| **Qwen3-Coder-Next** | 코드 생성 | 코딩 요청, 스크립트 작성 |
| **Qwen3-Coder-480B** | 대형 코드 | 대규모 리팩토링, 복잡 알고리즘 |
| **Qwen3-Next-80B** | 일반 텍스트 | 중급 분석 |
| **Qwen3-Coder-30B** | 빠른 코드 | 짧은 코드 스니펫 |
| **gpt-oss-20b** | 빠른 응답 | 50자 이내 간단 Q&A |

**판단 기준** (`router.py` 내장):
- 이미지 첨부 → VL 모델
- "PPT", "보고서" 키워드 → GLM-5
- "코드", "함수", 80자+ 길이 → Coder 계열
- 단순 질문 → 작은 모델 (응답 빠름)
- 복잡 신호 2개+ → 대형 모델

### 3.2 스킬 매칭 (자동 도구 선택)

397개 스킬을 다음과 같이 분류:

| 카테고리 | 예시 | 갯수 |
|---|---|---|
| **데이터 분석** | exploratory-data-analysis, polars, matplotlib | 80+ |
| **코딩 개발** | agent-python-pro, debugging, code-review | 50+ |
| **ML/AI** | scikit-learn, pytorch-lightning, statistical-analysis | 30+ |
| **보고서/문서** | pptx, docx, markdown-mermaid-writing, md-to-html | 20+ |
| **과학/논문** | literature-review, citation-management, pubmed-database | 30+ |
| **생물정보** | biopython, scanpy, pydeseq2 | 25+ |
| **화학** | rdkit, molfeat, deepchem | 15+ |
| **메타 스킬 (행동 가이드)** | brainstorming, systematic-debugging, verification-before-completion | 8+ |
| 기타 | knowledge-search, weekly-report, fab-* | 130+ |

사용자 쿼리에서 **트리거 단어** 자동 감지 → 스킬 자동 활성화. 또는 사용자가 사이드바에서 **수동 토글**도 가능.

### 3.3 도메인 지식 검색 (BM25)

사용자가 등록한 사내 문서 (보고서·메뉴얼·코드·아키텍처 등) 를 자동 검색:
- BM25 + 부분문자열 매칭 + 프론트매터 태그 매칭
- 등록 즉시 mtime 기반 자동 재인덱싱
- 한글 토큰화 지원

### 3.4 폴백 체인 (장애 대응)

모델 1개가 다운되거나 rate limit 에 걸리면 **자동으로 다음 모델로 전환**:

```
GLM-5 (1순위) → 429 Rate Limit
   ↓
Qwen3-Coder-Next (2순위) → 정상 응답
   ↓
사용자에게 답변 (지연 1-2초, 에러 없음)
```

UI 에 "⚠️ 대체: GLM-5 → Qwen3-Coder-Next" 알림 표시.

### 3.5 멀티에이전트 병렬 실행

여러 스킬이 선택되면 그룹으로 나눠 **동시 실행**:

```
스킬 4개: [matplotlib, polars, scikit-learn, statistical-analysis]
                    ↓
           그룹 1               그룹 2
   [matplotlib, polars]   [scikit-learn, statistical-analysis]
           ↓                       ↓
       Agent A (병렬)         Agent B (병렬)
           ↓                       ↓
                    합성 → 최종 답변
```

순차 실행 대비 **2-4배 응답 속도 향상**.

## 4. 사용자 입장 — 어떻게 쓰는가

### 4.1 가장 단순한 사용 (자동 모드)

**그냥 한국어로 질문하세요.**

```
✅ "이 CSV 파일 분석해줘" → 자동: 데이터 분석 스킬 + GLM-5 모델
✅ "주간보고 PPT 만들어줘" → 자동: weekly-report + pptx + GLM-5
✅ "이 코드 디버깅해줘" → 자동: debugging + code-review + Coder
✅ "M14 아키텍처 알려줘" → 자동: knowledge-search + 사내 문서 첨부
```

하네스가 알아서 처리.

### 4.2 수동 제어 (사이드바)

상세 제어가 필요하면:

| 사이드바 영역 | 기능 |
|---|---|
| **스킬 조합** | 4가지 미리 정의된 조합 선택 (📊 데이터 분석 / 💻 코딩 / 🤖 ML/AI / 📝 보고서) |
| **수동 스킬 선택** | 397개 중 직접 토글 |
| **모델 선택** | 드롭다운에서 직접 모델 지정 (자동 모드 OFF) |
| **자동 스킬 ON/OFF** | 자동 스킬 매칭 켜고 끄기 |

### 4.3 도메인 지식 등록

회사 내부 문서를 검색 가능하게 만들려면:
- "📚 내 지식" 메뉴 → 마크다운(.md) 형식으로 등록
- frontmatter 에 `tags`, `category`, `description` 입력
- 등록 즉시 검색 가능 (재시작 불필요)

권장 frontmatter 양식:
```yaml
---
title: "smartATLAS 주간 보고_20260429"
date: 2026-04-29
category: 주간보고
tags: [주간보고, smartATLAS]
description: "smartATLAS 주간 보고 4월 29일자"
owner: 사용자ID
---
```

## 5. 시나리오별 사용 예시

### 시나리오 1 — 데이터 분석
```
사용자: "이 CSV 파일에서 이상치 찾아줘"
첨부: data.csv

하네스 자동 처리:
- 모델: GLM-5 (CSV 첨부 + 분석 요청)
- 스킬: exploratory-data-analysis, polars, statistical-analysis
- 도메인 검색: 관련 컬럼 정의 자동 첨부
- 응답: 이상치 통계 + 시각화 코드 + 그래프 이미지
```

### 시나리오 2 — 보고서 PPT
```
사용자: "주간보고 PPT 만들어줘"

하네스 자동 처리:
- 모델: GLM-5 (PPT 생성 키워드)
- 스킬: weekly-report, pptx, scientific-writing
- 지식 검색: 지난 주간보고 .md 자동 첨부 (BM25)
- 응답: 답변 텍스트 + .pptx 다운로드 링크
```

### 시나리오 3 — 코드 디버깅
```
사용자: "이 함수 왜 무한루프 도는지 알려줘"
첨부: function.py

하네스 자동 처리:
- 모델: Qwen3-Coder-Next (코드 분석)
- 스킬: agent-debugger, systematic-debugging, code-review
- 메타 스킬: verification-before-completion (검증 가이드)
- 응답: 원인 분석 + 수정안 + 테스트 케이스
```

### 시나리오 4 — 사내 문서 검색
```
사용자: "M16HUB 아키텍처 알려줘"

하네스 자동 처리:
- 모델: GLM-5 (지식 검색 + 답변)
- 스킬: knowledge-search
- BM25 검색: knowledge/사용자/ 폴더에서 관련 .md 자동 매칭
- 응답: 매칭된 문서 본문 기반 답변 + 출처 명시
```

## 6. 모니터링 & 피드백

### 6.1 응답 메타정보 표시

매 답변 하단에 자동 표시:
```
[✅ 수동 스킬: knowledge-search, pptx, ...]
[모델: GLM-5] (127865자)
[⚠️ 대체: GLM-5 → Qwen3-Coder-Next]  ← 폴백 발생 시
[📝 형식: 보고서 / 스타일: 자동]
```

→ 어떤 모델·스킬·문자수로 답변했는지 투명하게 공개.

### 6.2 품질 점수 자동 평가

각 응답에 **A/B/C/D 등급** 자동 부여:
```
[QUALITY] score=83, grade=B, issues=[]
```

연속 C/D 등급 → 모델 자동 교체 또는 스킬 재선택.

### 6.3 세션 저장 / 복원

- 대화 자동 저장 (`/api/harness/session/save`)
- 서버 재시작 후에도 복원 가능
- 세션 목록 / 검색 / 삭제 UI 지원

## 8. 자주 묻는 질문 (FAQ)

### Q1. 왜 같은 질문이 어떤 때는 빠르고 어떤 때는 느린가요?
A. 하네스가 모델을 자동 선택하기 때문입니다. 단순 질문은 빠른 모델(gpt-oss-20b), 복잡 질문은 큰 모델(GLM-5/480B)로 분기합니다. 응답 메타에 표시된 모델을 확인하세요.

### Q2. 답변에 사내 문서가 안 첨부돼요.
A. (1) 사용자 폴더(`knowledge/사용자ID/`)에 문서가 등록되었는지 확인. (2) 문서의 frontmatter `tags`/`category` 에 검색어가 포함되었는지 확인. (3) 모델이 `knowledge-search` 스킬을 활성화했는지 응답 메타 확인.

### Q3. PPT 생성이 자꾸 실패해요.
A. (1) 보고서 조합 스킬(📝 보고서/문서) 사용. (2) 큰 데이터는 미리 요약 후 PPT 요청. (3) `weekly-report` 스킬로 정형 보고서 사용 권장.

### Q4. 특정 모델만 쓰고 싶어요.
A. 사이드바 모델 드롭다운에서 직접 선택. "자동 모드" 체크박스 OFF.

### Q5. 응답 품질이 낮아요. 어떻게 개선?
A. (1) 질문에 키워드를 명확히 (예: "분석" → "이상치 탐지"). (2) 관련 사내 문서를 미리 등록. (3) 응답 하단의 [QUALITY] 점수 모니터링. (4) 수동으로 메타 스킬(systematic-debugging, verification-before-completion 등) 추가.

### Q6. 한 번에 여러 작업 시키면?
A. 하네스가 자동으로 병렬 그룹화하지만, 매우 다른 영역(예: 데이터 분석 + 화학 시뮬)은 분리 요청 권장.

### Q7. 보안·외부 전송 걱정?
A. ASAS 는 폐쇄망 환경(SK Hynix 사내) 운영. 외부로 데이터 전송 없음. 모든 LLM 호출은 사내 HCP 서버 또는 로컬 GGUF 모델만 사용.

4. ✅ 자연어로 질문 — 그게 전부

7. ✅ 결과 만족스럽지 않으면 같은 질문에 키워드 추가하여 재시도
