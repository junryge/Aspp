하네스 기법, 쉽게 이해하기

AI한테 일을 시키는 "레시피"를
만들고 → 시험보고 → 고치는 반복 과정이에요

🍳 요리사 비유로 이해하기

요리사를 키운다고 생각해보세요. AI가 요리사, 스킬 문서가 레시피입니다.

요리사 훈련 과정 = 하네스 기법

1
레시피를 써준다 → "이렇게 만들어봐" →
2
실제로 만들어보게 한다 → 요리를 시킴 →
3
맛을 본다 → 잘 됐나? 안 됐나? →
4
레시피를 고친다 → "소금 좀 줄여봐" →
5
다시 만들어보게 한다 → 맛있을 때까지 반복! 🔄

📋 5단계로 보기

1

뭘 만들지 정하기

"AI야, 넌 이런 일을 해야 해"라고 스킬 문서를 작성해요.
언제 쓸 건지, 어떤 결과를 내야 하는지, 주의점은 뭔지 적어둡니다.

2

시험 문제 만들기

스킬이 잘 작동하는지 확인할 테스트 문제를 만들어요.
예: "이 PDF에서 표 뽑아줘", "이 데이터로 차트 만들어줘" 같은 진짜 사용자가 할 말들.

3

시험 보기 — 두 가지 버전

같은 문제를 두 가지 방법으로 풀게 해요.

구분설명
스킬 있을 때우리가 만든 레시피를 보고 풀기
스킬 없을 때아무 도움 없이 혼자 풀기

→ 둘을 비교하면 "스킬이 얼마나 도움이 되는지" 한눈에 보여요!

4

채점하기

두 가지 방법으로 채점합니다:

🤖 자동 채점

  • ✅ 파일이 만들어졌나?
  • ✅ 정답이 포함됐나?
  • ⏱ 걸린 시간은?

👁️ 사람이 직접 보기

  • 🎨 결과물이 보기 좋은가?
  • 💡 내가 원하는 느낌인가?
  • ✏️ 수정할 점이 있나?
5

고쳐서 다시 시험보기

채점 결과를 보고 스킬을 수정한 뒤, 다시 3~4단계를 반복합니다.
만족할 때까지. 보통 3~5번이면 충분해요.

🤔 왜 이렇게 해야 하나요?

"그냥 감으로 만들면 안 되나?" → 안 됩니다!

감으로 만들면 생기는 문제

"내 눈엔 괜찮은데?" → 실제 사용자에겐 아닐 수 있음
"고쳤더니 좋아진 거 같아" → 비교 데이터 없이는 증명 불가
"이게 효과가 있겠지?" → 숫자 없이는 확신 불가

정답률
