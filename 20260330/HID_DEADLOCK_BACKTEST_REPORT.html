2.3 Rule 2 — 속도 저하 (0.2%)

사실상 비활성 — 재설계 필요

전체 121,382 row에서 speed_ratio < 0.5인 건이 단 1건. speed_ratio 표준편차가 0.047로, 거의 모든 값이 0.93~1.06 구간에 집중되어 있어 현재 임계값(CRITICAL: 0.3, HIGH: 0.5)으로는 탐지가 불가능하다.

원인 추정

가능성 1: FREE_FLOW_SPEED가 실시간 주행 속도가 아닌 구간 설정 속도(기준 속도)일 수 있다. OHT 정체 시 이 값이 변하지 않고, 정체는 HID_VALUE 증가로만 반영될 수 있다.

가능성 2: OHT가 정체 시 이동이 없어 HID_INOUT 테이블에 기록되지 않을 수 있다. 이동 완료된 차량만 기록되므로, 정체 중인 차량의 속도 저하는 포착되지 않는다.

조치 제안

속도비 기반에서 유출 정지 기반으로 재설계: outflow_cnt = 0 (1분간 아무 차량도 출발하지 않음)을 정체 판단 대리 지표로 활용.

2.4 Rule 3 — 유입/유출 불균형 (67.2%)

결과

Rule 1 다음으로 강력한 탐지 메커니즘. WARNING 이상에서 평균 net_inflow는 3.6으로, 분당 3~4대씩 축적되는 패턴이 주요 경고 원인이다.

원인

특히 outflow_cnt = 0과 결합된 경우(CRITICAL 4점) IMMINENT 판정 2건 모두에서 발동했다. 유입은 있는데 유출이 완전히 정지된 상태가 데드락의 가장 강력한 전조 신호임을 확인.

2.5 Rule 4 — 정체 지속성 (0.1%)

사실상 비활성 — 재설계 필요

5건만 발동. 트리거 조건이 saturation ≥ 0.85 AND speed_ratio < 0.5의 동시 충족인데, speed_ratio < 0.5가 거의 발생하지 않으므로 함께 무효화. Rule 2와 동일한 근본 원인(속도비 안정성)에 기인.

조치 제안

속도비 조건을 제거하고, saturation ≥ 0.70 AND net_inflow ≥ 2가 N분 연속으로 변경하여 포화도+유입 축적의 시간적 지속을 판단하도록 재설계.

2.6 Rule 5 — 연쇄 데드락 (31.1%)

결과

공간적 확산 패턴 탐지에 유효. 인접 HID 그래프에서 평균 인접 8.7, 최대 인접 20(HID 110)으로 충분한 연결성 확인.

원인

HID 37~41 클러스터에서 집중 발동. 5개 HID가 모두 서로 인접하여, 하나의 포화가 인접 구역으로 전파되는 패턴이 명확히 관찰됨.

야간 본격 (0~7시)

WARNING 401건 (93.3%). 2~6시 구간에 피크 집중.

원인: 야간 배치 작업 집중, 운영 인력 최소 상태에서 자동 디스패치가 부하 분산 없이 작동, 교대 전환 시점의 밀린 작업 일괄 처리.

시간대 패턴 종합 원인

야간 배치 작업: 야간에는 대규모 배치 이동이 집중된다. 특정 공정 완료 후 다량의 Lot이 동시에 이동을 시작하면 특정 HID 구간에 유입이 급증한다.

교대 시간 효과: 0시 전후(야간 교대 전환)와 6~7시(주간 교대 전환)에서 스케줄러가 밀린 작업을 한꺼번에 디스패치하는 경향이 있다.

2~6시 피크: FAB 설비 가동률이 높으면서 물류 관리 인력이 최소인 구간. 자동 디스패치가 최단 경로로만 OHT를 배정하여 HID 37~41 클러스터에 집중.

6. 개선 방향

즉시 조치 (Week 1)

Rule 2 재설계: 속도비 → 유출 정지 기반 (outflow_cnt=0 AND inflow_cnt≥2 → CRITICAL)

Rule 4 재설계: 속도비 조건 제거 → saturation≥0.70 AND net_inflow≥2가 N분 연속

임계값 하향: sat_critical 0.95→0.80 / sat_high 0.85→0.65 / sat_medium 0.70→0.50

중기 조치 (Week 2~3)

FREE_FLOW_SPEED 의미 확인 / HID 37~41 FAB 레이아웃 위치 확인 / 야간 전용 임계값 설정

장기 조치 (Week 4~)

수정된 룰베이스로 2~4주 라벨 축적 → LightGBM 베이스라인 ML 모델 전환 → 5~10분 사전 예측

7. 결론

14시간 야간 데이터(985,144건)에 대한 룰베이스 백테스트 결과, 5개 룰 중 3개(포화도, 유입유출, 연쇄)가 유효하게 작동하며, 2개(속도, 지속성)는 데이터 특성상 재설계가 필요하다.

가장 주목할 발견은 HID 37~41 클러스터로, 이 5개 인접 구역이 전체 경고의 62%를 차지하며 야간 0~7시에 집중 발생한다. 이는 FAB 내 특정 물류 병목 지점의 존재를 시사하며, 해당 구간의 물리적 위치 확인과 함께 OHT 디스패치 로직의 부하 분산 개선이 필요하다.

임계값 조정과 Rule 2/4 재설계를 통해 WARNING 비율을 현재 0.35%에서 목표 0.5~2.0%로 끌어올린 뒤, 2~4주간 라벨을 축적하여 ML 예측 모델로 전환하는 것을 권장한다.
