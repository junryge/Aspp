1번 (전체 구조도):

최상단: LLM 3개 엔드포인트 (hcp 6모델, common 4모델, summary 1모델) + token.txt 인증
중간: Flask 서버 — 쓰는 길(ingest/query) + 읽는 길(BM25) + lint + 자동갱신 + REST API + 웹 UI 6탭

2번 (데이터 흐름도):

ingest (왼쪽 상단, coral): 텍스트 투입 → LLM 추출 → .md 생성 → index/concepts/log/BM25 자동 갱신
search (오른쪽 상단, blue): 검색어 → BM25+메타 스코어링 → 결과 반환
query (오른쪽 중간, purple): 질문 → BM25 검색 5개 → LLM에 전달 → 종합 답변
