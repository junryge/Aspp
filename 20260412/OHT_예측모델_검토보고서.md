## 3. OHT 경로 선택 방식 (현재 시스템)

현재 OHT 시스템은 **실시간 최적 경로 탐색** 방식으로 운영됩니다.

```
  ┌──────────────────────────────────────────────────┐
  │                                                    │
  │   작업 발생  →  현재 도로 상황 확인  →  최적 경로 결정  │
  │                                                    │
  │   * 자동차 내비게이션과 동일한 원리                    │
  │   * "지금 이 순간" 가장 빠른 길을 찾아줌               │
  │   * 상황이 바뀌면 경로도 즉시 변경                     │
  │                                                    │
  └──────────────────────────────────────────────────┘
```

경로 비용은 아래 요소들로 **실시간 계산**됩니다:

| 비용 요소 | 설명 |
|-----------|------|
| 구간 통과 시간 | 레일 길이와 현재 속도 기반 |
| 해당 구간의 대기 차량 수 | 차량이 많을수록 비용 증가 |
| 해당 구간의 작업 차량 수 | 작업 중인 차량은 더 큰 비용 |
| 대기 중인 물류 작업 수 | 작업이 몰린 구간은 비용 증가 |

**1,033대가 동시에 움직이면서** 각 구간의 상황이 매 순간 바뀌고,
그에 따라 최적 경로도 계속 바뀝니다.

## 4. 예측 가능 여부 검토 결과

### 4.1 전체 흐름 예측 → 가능

시간대별 전체 물류 패턴은 **현재 데이터만으로도 예측 가능**합니다.

| 예측 항목 | 예시 | 활용 |
|-----------|------|------|
| 시간대별 물동량 | "14시에 물동량이 가장 많다" | 인력/자원 배치 |
| 스테이션 처리량 | "스테이션A는 시간당 평균 80건 처리" | 병목 사전 감지 |
| 운반 소요시간 | "A→B 구간은 평균 6.2분 소요" | 납기 추정 |
| 정체 발생 구간 | "오후에 특정 구간 정체 빈번" | 차량 재배치 |

> FAB 생산 계획이 매일 유사하므로, 과거 패턴 기반 예측이 유효합니다.

### 4.2 개별 차량 위치 예측 → 불가능

"10분 후 특정 차량이 어디 있을까"는 예측할 수 없습니다.

| 원인 | 설명 |
|------|------|
| 차량 간 상호작용 | 1,033대가 서로 영향을 주고받아 결과를 예측할 수 없음 |
| 경로 수시 변경 | 경로 비용이 실시간으로 바뀌어 같은 출발-도착이라도 매번 다른 길 |
| 작업 발생 불확실성 | 새로운 물류 작업이 언제 발생할지 사전에 알 수 없음 |

## 5. 10분 / 20분 후 시뮬레이션

### 5.1 현재 상태: 불가

현재 시스템의 경로 탐색은 **"지금 이 순간"의 최적 경로**를 찾는 것이며,
미래 시점의 상황을 예측하는 기능은 아닙니다.

또한, 현재 수집 데이터에 **작업 명령 정보(CommandId)가 누락**되어 있어
차량이 어떤 작업을 수행 중인지 정확히 파악할 수 없습니다.

### 5.2 시뮬레이션이 가능해지려면

아래 **2가지를 추가 확보**하면, 10분~20분 후 상황을 시뮬레이션할 수 있습니다.

```
  ┌──────────────────────────────────────────────────┐
  │                                                    │
  │   필요 항목 ①: 레이아웃 데이터 (layout.xml)         │
  │                                                    │
  │   - FAB 내 모든 주행 경로(약 9,400개 노드)의         │
  │     좌표와 연결 관계                                 │
  │   - 시스템에 이미 존재하는 파일                       │
  │   - 제공만 받으면 즉시 활용 가능                      │
  │                                                    │
  └──────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────┐
  │                                                    │
  │   필요 항목 ②: 작업 명령 데이터 (CommandId)          │
  │                                                    │
  │   - 현재 데이터 수집 시 21개 항목만 수집 중            │
  │   - 원래 23개 항목이며, 22번째가 CommandId             │
  │   - 수집 설정 변경으로 확보 가능                      │
  │   - 수 주간 축적 후 작업 발생 패턴 분석 가능           │
  │                                                    │
  └──────────────────────────────────────────────────┘
```

### 5.3 시뮬레이션 예상 정확도

2가지를 확보한 후 시뮬레이터를 구축하면:

| 예측 시점 | 정확도 | 설명 |
|-----------|--------|------|
| 10분 후 | 높음 | 대부분의 차량이 현재 작업을 수행 중이므로 큰 변화 없음 |
| 20분 후 | 중간 | 새 작업 발생으로 차량 재배치 시작, 오차 증가 |
| 30분 이후 | 낮음 | 누적 오차가 커져서 실제와 괴리 발생 |
