## 검색 동작 방식

### 1단계: 키워드 추출
- 사용자 질문에서 불용어(은/는/이/가 등)를 제거하고 키워드 추출
- 1자 이상 키워드 허용 (FAB 코드, 층수 등 짧은 키워드 지원)

### 2단계: 매칭 및 점수 산정
| 매칭 유형 | 점수 | 설명 |
|-----------|------|------|
| 파일명 매칭 | **10점**/키워드 | "M14" → FAB_M14_컬럼.md |
| 내용 빈도 | **1~10점**/키워드 | 키워드 출현 횟수 반영 (최대 10회) |
| 구문 매칭 | **+15점** | 연속 키워드 조합이 같이 나올 때 |
| 전체 구문 | **+20점** | 질문 전체가 문서에 포함될 때 |
| 유사 한글 | **+3점** | 률↔율, 렬↔열 자동 변환 매칭 |
| 공백 무시 | **1~5점** | "디스크사용율" ↔ "디스크 사용률" |

### 3단계: 결과 전달
- 상위 3개 문서의 내용을 LLM 시스템 프롬프트에 주입
- LLM이 문서 기반으로 답변 생성

---

## 응답 규칙 (반드시 지켜라)

1. **문서에 있는 내용만 답변하라.** 없는 내용을 지어내지 마라.
2. **출처를 명시하라.** 어떤 파일에서 찾았는지 파일명을 표기하라.
3. **컬럼 질문 시 테이블 형태로 답변하라.** 컬럼명, 한글 설명을 포함.
4. **여러 FAB에 같은 컬럼이 있으면 비교 표로 정리하라.**
5. **문서에서 찾지 못한 경우 솔직하게 "해당 정보를 찾지 못했습니다"라고 답하라.**
