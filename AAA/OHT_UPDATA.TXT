원인 요약
진입 조건	엣지 집계	버퍼 플러시	마스터 업데이트
수정 전	VHL_CNT만 체크	무조건 실행	무조건 실행	무조건 실행
수정 후	VHL_CNT || HID_INOUT	if (HID_INOUT)	if (HID_INOUT)	if (HID_INOUT)
VHL_CNT=TRUE인 fab(예: M14A)에서 _calculatedVhlCnt()에 진입하면, 메서드 안에서 HID_INOUT 스위치를 체크하지 않고 엣지 집계/플러시/마스터 업데이트를 전부 실행했습니다.

즉, HID_INOUT=FALSE로 설정해도 VHL_CNT=TRUE이기만 하면 HID_INOUT 관련 데이터가 DB에 들어간 겁니다.

수정 내용
functionItem을 메서드 파라미터로 넘기도록 변경
VHL_CNT 코드는 if (VHL_CNT) 가드
HID_INOUT 코드 3곳 전부 if (HID_INOUT) 가드 추가
이제 HID_INOUT=FALSE면 해당 연산이 완전히 스킵됩니다.
