04

통합 코드

세 경로를 함수로 묶은 것. PDF는 페이지마다 디지털이면 텍스트 / 스캔이면 이미지로 자동 분기.

client.py
import fitz, base64, requests

URL = "http://<엔드포인트>/v1/chat/completions"

def post(messages):
    r = requests.post(URL, json={"model":"gemma-4-31B-it",
                                 "messages":messages})
    return r.json()["choices"][0]["message"]["content"]

def ask_text(q):
    return post([{"role":"user","content":q}])

def ask_image(b64, q):
    return post([{"role":"user","content":[
        {"type":"image_url",
         "image_url":{"url":f"data:image/png;base64,{b64}"}},
        {"type":"text","text":q}
    ]}])

def ask_pdf(path, q, dpi=200):
    doc, out = fitz.open(path), []
    for i, page in enumerate(doc):
        txt = page.get_text().strip()
        if txt:                    # 디지털 → 텍스트
            out.append(ask_text(f"{q}\n\n[p{i}]\n{txt}"))
        else:                      # 스캔/도면 → 이미지
            png = page.get_pixmap(dpi=dpi).tobytes("png")
            b64 = base64.b64encode(png).decode()
            out.append(ask_image(b64, f"{q} (p{i})"))
    return out

TL;DR

한 줄 정리

  • 텍스트 → 그냥 요청. quant·mmproj 신경 X.
  • 이미지 → base64로 content 배열에 실어서.
  • PDF → 페이지를 PNG로 렌더 후 이미지 요청. 단 디지털 PDF는 get_text()가 낫다.
  • vision 안 되면 → 서버 mmproj 문제. 클라이언트에서 못 고침.
