## ★ 도메인 물리구조 (해석의 근거)

### 핵심 사상 — HUBROOM은 단일 관문(single choke point)
**모든 M14↔M16 물류가 HUBROOM(3F) 하나를 반드시 통과한다.** 그래서 HUBROOM이 막히면 전체가 막히고, hot_area가 M16HUB로 쏠린다. (우연 아님 — 물리 구조)

### 건물·층
| 영역 | 층 | 비고 |
|---|---|---|
| M16 HUBROOM | 3F | 단일 관문 |
| M16A | 6F | 메인 |
| M16B | 10F | |
| M16EUV / M16WT | 2F | (2F에 둘 다) |
| M14A | 3F | CNV로 허브 연결 |
| M14B | 7F | LFT로 허브 연결 |

### 장치ID·이동방향 매핑 (★숫자→물리 번역기)
| 데이터 패턴 | 이동 방향 | 물리 의미 |
|---|---|---|
| `M14CNV` / `M14A.CNV...TO_M16HUB` | M14A CNV → M16HUB | **M14A 유입** (주 유입로) |
| `4ABLD...7F_TO_4F` / `M14LFT` | M14B(7F) → 허브 | **M14B 유입** |
| `6ABL...6F_TO_3F` | M16A(6F) → 허브(3F) | **M16A 유입** |
| `6ABL...3F_TO_6F` / `M16LFT` | 허브(3F) → M16A(6F) | **M16A 유출** |
| `6ABL...2F_TO_3F` | 2F → 허브 | **2F 유입** |
| `6ABL...3F_TO_2F` / `M16LFT2F` | 허브 → M16 2F(EUV/WT) | **2F 유출** |
| `M16HUBOHT` | 허브 → M16 2F/6F | OHT 분배 |

> ★ `6ABL`=M16A 리프터, `4ABLD`=M14B 리프터, `4AFC/CNV`=M14A 컨베이어, 끝자리 숫자(0111/0112)=리프터 호기.
>
> ★★ 장치 종류를 반드시 명시한다 ("수직반송" 같은 모호한 말 금지):
> - **LFT(리프터)** = 층간 반송 (예: 6ABL, 4ABLD) → "리프터 정체"
> - **CNV(컨베이어)** = 같은 층 반송 (예: 4AFC, M14A↔허브) → "컨베이어 정체"
> - **OHT** = 레일 반송 (VHL 제어) → "OHT 정체"
> → 컬럼 패턴으로 장치를 판별해 **"어느 장치(리프터/컨베이어/OHT)가 막혔다"** 로 정확히 쓴다.

## ★ 세분화 해석 규칙 (리프터막힘 = RC 가 떴을 때)

지금까지 "M16HUB 리프터막힘"으로 뭉뚱그렸다면, 이제 **propagation_chain·signals·전파경로의 리프터ID/방향**으로 쪼개서 짚는다:

1. **어느 리프터** — 6ABL0112 / 6ABL0122 등 호기 단위로
2. **어느 방향** — `X_TO_3F`(유입) vs `3F_TO_X`(유출)
3. **물리 경로로 번역** — 위 매핑표로 "허브↔M16A 유출 막힘" 식으로

**예시 문장:**
- ❌ (기존) "M16HUB 리프터 정체"
- ✅ (도메인) "**M16A 리프터(6ABL0112)가 허브→6F 유출 방향**으로 정체 — M16A로 못 올려보내 허브에 쌓임"

## ★ 차등화 — 유입 vs 유출 구분

정체 원인을 **방향으로 차등 해석**한다 (대응이 다름):

| 구분 | 패턴 | 의미 | 대응 |
|---|---|---|---|
| **유입 정체** | `X_TO_3F`, `..TO_M16HUB` | 허브로 **들어오는** 물량 막힘 | 상류(M14A·M16A) 유입 제어 |
| **유출 정체** | `3F_TO_X`, `M16LFT` | 허브에서 **나가는** 쪽 막힘 | 하류(M16A 6F) 저장·리프터 확인 |

> ★ 유출이 막히면 → 허브 STB 포화(RD)로 이어짐(back-pressure). 유입이 막히면 → 상류 Queue 누적(RB).
> 즉 **RD·RC가 같이 뜨면 "유출 막힘", RB가 앞서면 "유입 폭주"** 로 구분.
