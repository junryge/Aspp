TD-MPC · Learning World Model

복원하지 않는다,
예측한다 — TD-MPC 월드모델

TD-MPC(Temporal Difference Learning for Model Predictive Control)는 세계의 원본을 되그리는 대신, 미래의 잠재 상태와 보상만을 예측하는 모델을 학습하고 그 압축된 세계 안에서 최적의 행동을 계획하는 모델 기반 강화학습 알고리즘입니다. OHT 월드모델이 “규칙”에서 “학습”으로 진화하기 위해 채택한 기반 기술을, 이 문서 한 장으로 설명합니다.

Core Idea

한 문장으로 이해하기

“세계를 통째로 기억하는 대신, 결정에 필요한 것만 남긴 ‘요약된 세계’를 배우고, 그 안에서 여러 미래를 미리 살아본 뒤 가장 좋은 행동을 고른다.”

바둑 기사가 수를 읽는 방식과 같습니다. 기사는 머릿속에서 바둑판 사진을 픽셀 단위로 그리지 않습니다. 형세라는 압축된 표현으로 “이 수를 두면 → 상대는 이렇게 → 그러면 형세가 이렇게”를 몇 수 앞까지 굴려 보고, 가장 유리해지는 수를 둡니다.

TD-MPC가 정확히 이 구조입니다.

  • 형세 = 잠재 상태(latent state) — 원본 세계를 결정에 필요한 만큼만 압축한 벡터
  • 수읽기 = 잠재 동역학(latent dynamics) — “이 행동을 하면 다음 형세는 이것”을 예측
  • 유불리 판단 = 보상(reward)과 가치(value) 예측
  • 최선의 수 선택 = MPC 계획(planning) — 후보 행동 궤적 여러 개를 굴려 보고 최고를 선택

이름의 두 축이 그대로 알고리즘입니다. MPC(Model Predictive Control)가 “모델로 미래를 굴려 보고 행동을 고르는” 몸통이고, TD(Temporal Difference) 학습이 “굴려 본 지평선 너머의 가치까지 평가하는” 눈입니다.

Design Choice

왜 복원(reconstruction)하지 않는가

Dreamer 계열 등 많은 월드모델은 잠재 상태에서 원본 관측을 다시 그려내는(복원) 과제를 함께 학습합니다. 복원이 잘 되면 잠재 상태가 정보를 충분히 담았다는 보증이 되기 때문입니다. 하지만 복원에는 비용이 있습니다 — 모델이 결정과 무관한 디테일까지 기억하도록 강제된다는 점입니다. 배경 무늬, 조명, 지금 결정에 아무 영향 없는 구석의 상태까지 전부요.

TD-MPC는 복원 과제를 버리고, 잠재 상태에게 딱 세 가지 질문에만 답하게 합니다.

  1. 이 행동을 하면 다음 잠재 상태는 무엇인가? (동역학 일관성)
  2. 이 행동의 보상은 얼마인가?
  3. 이 상태의 장기 가치는 얼마인가?

즉 잠재 공간이 “제어에 필요한 정보”만 남기고 나머지를 버리도록 학습됩니다. 모델은 작아지고, 학습은 빨라지고, 노이즈에 강해집니다.

OHT에 이것이 결정적인 이유: 우리의 “관측”은 차량 수백 대 × 노드 수천 개의 거대한 그래프 상태입니다. 이걸 매 스텝 복원하는 것은 낭비를 넘어 불가능에 가깝습니다. 그런데 우리가 정말 알고 싶은 것은 “이 조치를 하면 10분 뒤 큐·OBS·반송시간이 어떻게 되나” — 정확히 보상과 미래 요약 상태입니다. TD-MPC의 설계 철학과 문제의 요구가 일치합니다.

Architecture

다섯 개의 부품 — 전부 신경망입니다

TD-MPC는 다섯 개의 학습 가능한 부품으로 이루어집니다. 그림으로 먼저 보면:

부품기호하는 일
① 인코더z = h(s)거대한 원본 관측 s를 작은 잠재 상태 z로 압축. 우리 적용안에서는 GNN(그래프 신경망)으로 구현
② 잠재 동역학z' = d(z, a)“상태 z에서 행동 a를 하면 다음 상태는 z'” — 월드모델의 심장. 원본으로 돌아가지 않고 잠재 공간 안에서만 전진
③ 보상 예측기r̂ = R(z, a)그 한 스텝의 좋음/나쁨을 숫자로 예측
④ 가치 함수Q(z, a)“여기서부터 끝까지 잘하면 얼마나 좋아지나” — 계획이 못 보는 지평선 너머를 담당. TD 학습으로 훈련
⑤ 정책a = π(z)계획의 출발점이 되는 좋은 행동 후보를 즉시 제안 (계획을 데운다)

다섯 부품은 따로 훈련되지 않고 공동 학습됩니다 — 실제 경험 데이터에서 “동역학이 예측한 잠재 상태가 실제와 일치하는가, 보상 예측이 맞았는가, 가치가 TD 목표와 일치하는가”를 하나의 손실 함수로 묶어 함께 최적화합니다.

Planning

잠재 공간에서 계획하기 — 수백 개의 미래를 살아본다

매 순간 행동을 정할 때, TD-MPC는 정책이 내놓는 답 하나를 그냥 쓰지 않습니다. MPPI(Model Predictive Path Integral)라는 샘플링 계획법으로 더 나은 답을 찾습니다.

  1. 후보 생성 — 앞으로 H스텝 동안의 행동 궤적 후보를 수백 개 샘플링합니다 (일부는 정책 π의 제안으로 시작).
  2. 미래 시뮬레이션 — 각 후보를 잠재 동역학 위에서 굴려, 예측 보상의 합 + 마지막 상태의 가치 Σ r̂ + Q(z_H)로 점수를 매깁니다.
  3. 정제 — 점수 상위 후보들의 분포로 샘플링 분포를 갱신하고, 몇 차례 반복해 궤적을 다듬습니다.
  4. 한 걸음만 실행 — 최선 궤적의 첫 행동만 실제로 수행하고, 다음 순간 처음부터 다시 계획합니다.

“한 걸음만 실행하고 다시 계획”이 MPC의 핵심입니다. 세계가 예측과 조금 달라져도 다음 스텝에서 즉시 보정되므로, 모델이 완벽하지 않아도 견고하게 동작합니다. 이 모든 시뮬레이션이 압축된 잠재 공간에서 일어나므로, 수백 개의 미래를 굴려도 실시간이 가능합니다.

Learning

TD 학습 — 지평선 너머를 보는 눈

계획은 H스텝(예: 몇 분)까지만 내다봅니다. 그런데 OHT의 데드락처럼 수십 분에 걸쳐 쌓이는 결과는 지평선 밖에 있습니다. 이걸 담당하는 것이 가치 함수 Q이고, Q를 훈련하는 방법이 시간차(Temporal Difference) 학습입니다.

# TD 목표: "지금의 가치 = 한 스텝 보상 + 다음 상태의 가치"
Q(z_t, a_t)  ←  r_t + γ · Q(z_t+1, π(z_t+1))
# 예측과 목표의 차이(시간차 오차)를 줄이는 방향으로 반복 갱신

직관적으로는 “미래의 평가를 조금씩 현재로 끌어당기는” 과정입니다. 데드락이라는 나쁜 결말의 신호가, 학습을 거듭할수록 그 몇십 분 전 상태들의 가치 평가 속으로 스며듭니다. 그래서 계획기가 몇 스텝만 굴려 봐도, 마지막에 붙는 Q값이 “그 방향은 결국 데드락으로 간다”고 경고해 줄 수 있게 됩니다 — 우리가 룰로 만들었던 ‘전조 감지’를, 모델이 데이터에서 스스로 배우는 것입니다.

Application

OHT에 어떻게 적용하나 — 네 가지 설계 결정

상태 → GNN 인코더

OHT의 세계는 애초에 노드·엣지 그래프입니다. 이를 벡터로 뭉개 MLP에 넣으면 가장 큰 구조 정보를 버리는 셈이므로, 인코더 h는 GNN(그래프 신경망)으로 구현합니다. 레일 토폴로지·차량 분포·구간 혼잡이 구조 그대로 잠재 상태에 담깁니다.

행동 → 매크로 손잡이

차량 수백 대의 개별 제어는 조합 폭발합니다. 행동 공간은 HID 구역별 진입 상한, 리프터 할당률, 구간 LineCost 가중치 같은 매크로 수준의 조절 손잡이로 정의해 차원을 낮춥니다. 이 손잡이가 실제로 유효한지가 1차 검증 대상입니다.

보상 → 검증된 전조 지표 재활용

“데드락 = 큰 벌점”만으로는 신호가 너무 희소합니다(발생 자체가 드묾). 이미 실 사건으로 검증한 전조 지표들 — OBS 추세, 반송시간, FAB 간 큐, 리프터 처리량 역전 — 을 형태화된 보상(shaped reward)으로 씁니다. 기존 R-A/R-B/R-C 룰이 여기서 다시 자산이 됩니다.

훈련장 → WORLD_SIM

실제 FAB에서 강화학습 실험은 불가능합니다. 대신 이미 구축·검증된 WORLD_SIM (그래프 시뮬레이션 + 실 로그 리플레이 + Dijkstra/LineCost 경로)을 gym 스타일 환경으로 감싸 안전한 훈련장으로 씁니다. 지금까지의 작업이 전부 학습 인프라가 됩니다.

설명 가능성 원칙: 잠재 공간 모델은 블랙박스이므로, 기존 룰 기반 3단계 경보는 폐기하지 않고 병행합니다. 학습 모델의 안전망이자 성능 비교의 베이스라인입니다 — 학습형이 룰 기반을 명확히 이기기 전까지 운영의 기준은 바뀌지 않습니다.

Comparison

규칙 기반 vs 학습형 — 무엇이 달라지나

현재 — 규칙 기반 WORLD_SIM

사람이 세계의 법칙을 적는다

  • 경로: Dijkstra + LineCost (실제 시스템과 동일 수식)
  • 데드락: Tarjan SCC로 순환 대기를 수학적 판정
  • 경보: 실 사건으로 검증된 R-A/R-B/R-C 룰
  • 장점: 투명함 — 왜 경보가 울렸는지 근거를 말할 수 있음
  • 한계: 새 FAB·새 패턴마다 룰을 손으로 재설계
다음 — 학습형 TD-MPC

모델이 세계의 법칙을 배운다

  • 동역학: 데이터에서 학습한 잠재 전이 모델
  • 전조: TD 학습이 나쁜 결말의 신호를 가치로 흡수
  • 행동: 잠재 공간 계획으로 개입까지 제안
  • 장점: 확장성 — 새 패턴을 데이터로 흡수, 예측→제어
  • 한계: 블랙박스 — 그래서 룰 기반과 병행 운영

둘은 대체 관계가 아니라 역할 분담입니다. 규칙 기반 시뮬레이터는 학습형의 훈련장이자 심판이 되고, 학습형은 규칙 기반이 못 가던 영역(미검증 패턴, 개입 제안)으로 전선을 넓힙니다.

Version Choice

TD-MPC2를 쓰는 이유

원조 TD-MPC(2022)의 후속인 TD-MPC2(2023)를 채택합니다. 알고리즘의 뼈대는 같지만, 실무 도입 관점에서 중요한 개선이 있습니다.

  • 단일 하이퍼파라미터 설정 — 원조는 태스크마다 튜닝이 필요했지만, TD-MPC2는 한 벌의 설정으로 수십 개 태스크에서 안정적으로 동작함이 입증되었습니다. 도메인 튜닝 부담이 큰 산업 적용에서 결정적인 장점입니다.
  • 안정화된 학습 — 이산화된 회귀(discrete regression) 기반 보상·가치 학습 등으로 스케일이 다른 보상에도 견고합니다. 큐(수천 건)와 반송시간(분)처럼 단위가 뒤섞인 우리 보상 설계에 유리합니다.
  • 모델 크기 확장성 — 모델을 키울수록 성능이 늘어나는 스케일링 특성이 확인되어, 데이터가 쌓일수록 함께 성장하는 로드맵을 그릴 수 있습니다.

Roadmap

도입 로드맵과 현재 위치

한 번에 전부 바꾸지 않습니다. 각 단계가 앞 단계의 검증을 통과해야만 다음으로 넘어갑니다.

1

환경 포장 — 훈련장 만들기

검증된 WORLD_SIM을 gym 스타일 환경(state / action / reward / step)으로 감쌉니다. 기존 시뮬레이터가 학습의 “세계”가 됩니다.

통과 조건 — 리플레이 재현 일치
2

모델 먼저, 맞대결

TD-MPC에서 계획(planning)을 뗀 잠재 동역학 + 보상 예측기만 실 로그로 학습해, 기존 매크로 예측기와 동일 벤치마크로 대결합니다. 기준: 2026-04-21 실 사건에서 13:50 경보(리드타임 10분)를 이기고, 16:00 거짓 경보를 똑같이 차단하는가.

통과 조건 — 기존 예측기 대비 우위 입증
3

계획 결합 — 예측에서 제어로

모델이 이긴 뒤에야 잠재 공간 계획을 붙여, 매크로 행동(진입 상한·할당·가중치)의 최적 궤적을 제안하는 제어 단계로 확장합니다.

최종 형태 — 개입을 제안하는 월드모델

현재 위치 — 훈련장까지 완성된 단계입니다. 현실을 복제한 검증된 시뮬레이션 환경(WORLD_SIM)이 학습형 월드모델이 마음껏 뛰고 넘어져도 되는 도장(道場) 역할을 합니다. 아직 모델이 이 안에서 훈련을 시작하지는 않았습니다.

조만간 조건이 맞춰지는 대로 — 행동 공간 정의, 보상 설계, 학습용 데이터 정비가 끝나는 대로 — 이 훈련장 위에서 월드모델을 학습 가능한 형태로 진화시킬 것입니다. 훈련장이 먼저, 선수는 그 다음. 순서는 이미 잡혀 있습니다.

Glossary

Temporal Difference Learning for Model Predictive Control. 잠재 상태와 보상만 예측하는 모델을 학습하고, 잠재 공간에서 행동을 계획하는 모델 기반 강화학습 알고리즘 (Hansen et al., 2022 / TD-MPC2는 2023).
월드모델
세계의 동작 법칙을 담아 “이 행동을 하면 어떻게 될까”를 시뮬레이션할 수 있는 모델. 규칙으로 적을 수도(현재 WORLD_SIM), 데이터에서 배울 수도(TD-MPC) 있습니다.
잠재 상태 (latent state)
원본 관측을 결정에 필요한 정보만 남기고 압축한 벡터. 바둑 기사의 “형세”에 해당합니다.
복원 (reconstruction)
잠재 상태로부터 원본 관측을 되그리는 학습 과제. TD-MPC는 이를 쓰지 않는 것이 특징입니다.
보상 (reward)
한 스텝의 좋음/나쁨을 숫자로 정의한 것. 강화학습이 최대화하려는 대상.
가치 함수 (Q)
“이 상태에서 앞으로 끝까지 잘하면 얼마나 좋은가”의 추정치. 계획 지평선 너머를 담당합니다.
TD 학습
Temporal Difference. “지금의 가치 = 한 스텝 보상 + 다음 상태의 가치”라는 관계로 가치 함수를 반복 갱신하는 학습법.
MPC
Model Predictive Control. 모델로 미래를 굴려 최선의 행동 궤적을 고르되, 첫 걸음만 실행하고 매 순간 다시 계획하는 제어 기법.
MPPI
Model Predictive Path Integral. 행동 궤적 후보를 확률적으로 샘플링하고 점수 상위 후보 쪽으로 분포를 좁혀 가는 샘플링 기반 계획법.
GNN
Graph Neural Network. 노드·엣지 구조를 그대로 학습하는 신경망. 레일 그래프인 OHT 세계의 인코더로 채택 예정.
gym 환경
강화학습 표준 인터페이스(state / action / reward / step). WORLD_SIM을 이 형태로 감싸면 어떤 RL 알고리즘이든 접속해 훈련할 수 있습니다.
