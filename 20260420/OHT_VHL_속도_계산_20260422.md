# OHT VHL 속도 계산 방법

> 작성일: 2026-04-22
> 대상 코드: `WORD_MODEL/WORLD_SIM/velocity_tracker.py`
> 관련 모듈: `world_model.py`, `data_loader.py`, `config.py`

OHS 시뮬레이터(WORLD_SIM) 에서 각 **OHT 차량(VHL)** 의 순간 속도(m/min) 를 계산하는 방법을 정리한다. 핵심 구현은 `velocity_tracker.py` 의 `VelocityTracker.update()` 에 있으며, UDP 메시지(`currentNode`, `nextNode`, `distance`) 와 수신 시각의 차분으로 차량별 시계열 속도를 만든다.

---

## 1. 입력 데이터

| 항목 | 출처 | 단위 | 비고 |
|---|---|---|---|
| `currentNode` | UDP / OHT raw CSV | int | 현재 세그먼트의 시작 노드 ID |
| `nextNode` | UDP / OHT raw CSV | int | 현재 세그먼트의 끝 노드 ID |
| `distance` | UDP / OHT raw CSV | **dm** (=100mm) | `currentNode` 로부터의 진행 거리 |
| `_time` | OHT raw CSV | datetime | UDP 수신 시각 (차량별) |
| `edge_dist_map[(a,b)]` | `layout_cache.json` | **dm** | `a→b` 엣지 전체 길이 |

> 거리 단위는 UDP/layout 모두 **dm (데시미터, 1 dm = 100 mm)** 로 통일되어 있다. 변환 상수 없이 바로 산술 가능.

---

## 2. 기본 공식

```
speed [m/min] = Δdistance_dm × 6 / Δt_sec
```

### 유도

- 1 dm = 0.1 m
- 1 분 = 60 초
- 따라서:
  `(Δdm × 0.1 m/dm) / Δt_sec × 60 s/min = Δdm × 6 / Δt_sec  [m/min]`

즉 **계수 6** 은 `0.1 × 60` 의 상수이다. (`velocity_tracker.py:115`)

---

## 3. 세 가지 계산 케이스

`prev.edge = (prev_curr, prev_next)`, 현재 `edge = (curr, next)` 로 두고 `Δdm` 계산 방식은 세 가지:

### Case A — 같은 엣지 내부 이동 (`prev.edge == edge`)
```
Δdm = distance - prev.distance
```
- `velocity_tracker.py:95-97`
- 가장 일반적인 경우. 같은 구간을 연속 통과 중.

### Case B — 자연스러운 엣지 전환 (`prev.edge[1] == edge[0]`)
```
prev_len = edge_dist_map[prev.edge]
Δdm     = (prev_len - prev.distance) + distance
```
- `velocity_tracker.py:98-101`
- 이전 엣지를 마저 끝내고 새 엣지로 넘어간 총 이동량.

### Case C — 불연속(샘플 누락) — BFS 경로 추정
```
path_len = BFS 최단경로 길이 (prev.edge[1] → edge[0])
prev_len = edge_dist_map[prev.edge]
Δdm     = (prev_len - prev.distance) + path_len + distance
```
- `velocity_tracker.py:103-110`
- UDP 드롭이나 프레임 점프로 엣지가 떨어진 경우.
- BFS 탐색 한도는 `BFS_MAX_HOPS = 20`. 경로 못 찾으면 측정 불가 처리.
- 결과는 `_bfs_cache` 에 저장해 같은 노드쌍 재조회 가속.

## 4. 안전 가드 / 필터링

| 조건 | 처리 | 위치 |
|---|---|---|
| 최초 관측 (`prev is None`) | `None` 반환, prev 저장 | 73-75 |
| 같은 시각 재호출 (`t == prev.time`) | 마지막 계산값 캐시 반환 | 77-79 |
| 과거 UDP 늦게 도착 (`t < prev.time`) | 캐시 반환, prev 갱신 X | 81-83 |
| `elapsed <= 0` | 캐시 반환 | 86-88 |
| `elapsed > MAX_GAP_SEC (30s)` | 측정 불가, prev 갱신 후 캐시 반환 | 91-93 |
| `Δdm < 0` (후진 / distance 리셋) | `speed = 0.0` | 112-113 |
| `NaN` / `inf` | 측정 불가, prev 갱신 후 캐시 반환 | 116-118 |
| 범위 벗어남 | `max(MIN_SPEED, min(MAX_SPEED, speed))` 클램프 | 119 |

### 상수
```python
MIN_SPEED    = 0.0     # m/min  (정지 허용; ATLAS는 1.5)
MAX_SPEED    = 300.0   # m/min  (= 18 km/h, 안전 상한)
MAX_GAP_SEC  = 30.0    # 이 이상 갭은 평균속도로 의미 없음
BFS_MAX_HOPS = 20
```
(`velocity_tracker.py:40-43`)

## 5. 차량별 상태 유지

`VelocityTracker` 는 차량 ID(`vid`) 별로 두 개의 맵을 보관:

- `self._prev[vid]: _PrevState(time, edge, distance)` — 마지막 UDP 스냅샷
- `self._last_speed[vid]: float` — 마지막 계산 속도(캐시)

`reset(vid)` 로 개별 차량만, `reset()` 로 전체 초기화 가능. 프레임 점프 / 시뮬레이션 리플레이 점프 시 호출.

---

## 6. 호출 지점 (연동)

`WORD_MODEL/WORLD_SIM/world_model.py:191-224` 의 `load_vehicles_from_frame()` 에서 매 프레임 호출:

```python
self.velocity_tracker = VelocityTracker(self.edge_dist_map, adj_map)
...
udp_time = vdata.get('_time') or frame_time
velocity = self.velocity_tracker.update(
    vid, udp_time, current_node, next_node, distance
)
# velocity 가 None 이면 측정 불가 — 가짜 값으로 채우지 않음 (Option B 정직화)
```

- 차량별 실제 UDP 도착 시각(`_time`) 을 우선 사용, 없으면 프레임 시각 fallback.
- `None` 을 반환하면 UI 는 "측정 불가" 로 표기 (기본 속도로 메우지 않음).

## 7. HID_INOUT `FREE_FLOW_SPEED` 와 차이

`LOGPRESSO_HID_INOUT_*.csv` 의 `FREE_FLOW_SPEED` 컬럼은 **구간(HID) 통과 시 기록된 자유 흐름 속도 값** 으로, HID 이벤트 단위의 집계/분석에 쓰인다.

| 항목 | VelocityTracker | HID_INOUT FREE_FLOW_SPEED |
|---|---|---|
| 주기 | UDP 프레임 단위 (0.5–1초) | HID 구간 통과 시마다 (비주기) |
| 대상 | 차량별 순간 속도 | 구간별 통과 속도 |
| 용도 | 시뮬레이션 / UI 표시 | 통계 집계 / 구간 분석 |
| 출처 | 위치 차분 계산 | 원본에 기록된 값 그대로 |

시뮬레이션 재생 UI 의 차량 속도 표시는 **반드시 VelocityTracker 값** 을 쓴다.

## 8. 예시 계산

### 예 1 — Case A (같은 엣지)
- `prev`: t=14:00:00.000, edge=(19,2), distance=100
- `now `: t=14:00:01.000, edge=(19,2), distance=250
- `Δt = 1.0s`, `Δdm = 150`
- **speed = 150 × 6 / 1.0 = 900 m/min** → 클램프 → **300 m/min (MAX)**

### 예 2 — Case B (엣지 전환)
- `prev`: t=14:00:00.000, edge=(19,2), distance=350 (엣지 길이 400 dm)
- `now `: t=14:00:01.000, edge=(2,5), distance=80
- `Δt = 1.0s`, `Δdm = (400-350) + 80 = 130`
- **speed = 130 × 6 / 1.0 = 780** → **300 m/min (MAX)**

### 예 3 — 정상 범위
- `Δdm = 30`, `Δt = 1.0s`
- **speed = 30 × 6 / 1.0 = 180 m/min** (≈10.8 km/h)

## 10. 요약

1. **공식**: `speed [m/min] = Δdm × 6 / Δt_sec`
2. **단위**: 거리 dm, 시간 s, 속도 m/min (MAX 300)
3. **케이스 3종**: 같은 엣지 / 엣지 전환 / BFS 보간
4. **가드**: MIN/MAX 클램프, MAX_GAP_SEC, 후진 0 처리, NaN 필터
5. **상태**: 차량별 `_prev`, `_last_speed`, BFS 결과 캐시
6. **연동**: `world_model.py:load_vehicles_from_frame()` 에서 매 프레임 호출, `None` 이면 UI 에 "측정 불가" 표기
7. **HID FREE_FLOW_SPEED 는 별개** 의 구간 통계 필드이며 차량 순간 속도와 혼용 금지
