PART I Definition 월드 모델이란 무엇인가

월드 모델의 정의

LLM이 "언어의 통계"를 학습했다면, 월드 모델은 "세계의 동역학"을 학습한다. 토큰 예측기에서 세계 예측기로의 진화.

월드 모델 = 언어(텍스트)가 아니라,
"세계 그 자체의 작동 방식"을 학습한 모델.
단어의 다음 토큰을 예측하는 게 아니라,
세계의 다음 상태를 예측한다.
Fig. 01AI 패러다임의 3단계 진화 — LLM은 이미 정점에 도달했고, 다음 물결은 월드 모델이다.

LLM vs 월드 모델 — 예측 대상의 차이

둘의 근본 차이는 "무엇을 예측하는가"에서 시작된다. LLM은 다음 토큰을, 월드 모델은 다음 상태를 예측한다.

구분 LLM World Model
예측 대상 다음 토큰 다음 상태 (물리/시각/행동 결과)
입력 텍스트 시퀀스 관측 + 행동(action)
목표 "그럴듯한 말" 생성 "그럴듯한 세계" 생성
학습 데이터 인터넷 텍스트 비디오, 센서, 행동 로그, 물리 데이터
인과성 상관(correlation) 중심 인과(causation) + 물리 법칙
검증 방법 인간 평가 실제 세계와 맞는가

PART II Why 왜 월드 모델을 해야 하는가

왜 지금,
월드 모델인가

LLM이 아무리 발전해도 구조적으로 풀 수 없는 문제들이 있다. 월드 모델은 그 벽을 넘는 유일한 경로다.

① LLM이 풀 수 없는 문제들

LLM은 본질적으로 "다음 토큰 예측기"다. 이 구조적 특성 때문에 아래 여섯 가지 문제를 원리적으로 풀 수 없다.

풀 수 없는 문제 왜 LLM이 못 푸는가
물리 법칙텍스트엔 중력·관성이 없음
공간 추론"왼쪽에서 세 번째"를 진짜로 이해 못함
인과 관계상관만 학습, 원인→결과 체인이 없음
장기 계획내부에서 "돌려볼" 세계가 없음
환각 제거사실을 검증할 기준 모델이 없음
실시간 제어행동(action) 개념 자체가 없음

② Physical AI 시대

로봇·자율주행·드론·팩토리 자동화는 LLM만으로는 작동하지 않는다. 예시 하나로 차이를 명확히 볼 수 있다.

// "컵을 집어라" 라는 지시가 주어졌을 때 LLM: "컵을 집어라" → 텍스트로 지시 이해 ✓ → 어떻게 집을지 모름 ✗ World Model: "컵을 집어라" → 세계 상태 인식 ✓ → 팔 움직임 시뮬레이션 ✓ → 컵이 떨어질지 예측 ✓ → 성공 경로 선택 ✓ // 엔비디아 젠슨 황의 "Physical AI" 핵심 엔진 = 월드 모델 // 휴머노이드 로봇, 자율주행, 스마트 팩토리의 필수 기반

③ 데이터의 한계 돌파

LLM은 인터넷 텍스트를 거의 다 먹어치웠다. 학습 데이터 고갈이 현실이다. 월드 모델은 완전히 다른 데이터 소스를 연다.

Fig. 03텍스트 고갈 시대 — 월드 모델은 비디오·센서·시뮬레이션·3D 공간이라는 훨씬 더 큰 데이터 지평을 연다.

④ 진짜 추론(Reasoning)을 하려면 필요하다

"추론한다" = 머릿속에서 시뮬레이션을 돌린다. 체스 고수는 "이 수를 두면 3수 뒤에…" 를 머릿속에서 돌린다. 엔지니어는 "이 설계를 바꾸면 온도가…"를 돌린다. 운전자는 "저 차가 끼어들면…"을 돌린다.

이것이 mental simulation이며, 이걸 하려면 월드 모델이 필수다. LLM의 Chain-of-Thought는 흉내일 뿐, 진짜 추론은 세계를 돌려보는 것이다.

⑤ 환각(Hallucination)의 근본 해결

LLM 환각의 원인은 "사실 여부를 체크할 기준 모델이 없음"이다. 월드 모델이 있으면 이렇게 검증이 가능하다.

// Grounded AI Pipeline LLM 생성 → "이 답변이 세계 모델과 일치하는가?" → 검증 // → Grounded AI (실제 세계에 근거한 AI)로 진화 // → 의료·법률·금융 같은 고신뢰 영역에서 쓸 수 있게 됨

⑥ AGI의 핵심 조건

대부분의 AI 연구자들이 동의하는 AGI의 조건: "행동하고, 예측하고, 계획하고, 상상할 수 있어야 한다." 이 네 가지를 다 하려면 월드 모델이 있어야 한다.

능력 LLM World Model
행동 (Act)✗✓
예측 (Predict)△ (텍스트만)✓ (세계 전체)
계획 (Plan)△ (겉흉내)✓ (진짜 rollout)
상상 (Imagine)△✓

⑧ 제조·물류·반도체 현장의 필연

특히 복잡한 물리 환경을 다루는 산업에서는 선택이 아닌 필수다.

  • 반도체 FAB — 수천 대 장비, 수만 개 반송 로직 → 월드 모델로 최적화
  • 자동차 공장 — 라인 전체 시뮬레이션
  • 물류 창고 — 로봇 수백 대 협업
  • 항공·철도 — 스케줄 + 물리 제약 결합

텍스트 LLM으로는 운영 자체가 불가능한 도메인이다. 월드 모델이 유일한 해법이다.
