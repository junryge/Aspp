```
[원본 데이터]                         [처리]                 [결과]
─────────────────────────────────   ───────────────────     ───────────────
M16A_BR_DATA/

 │                               │
 └─ 20260421.zip                 │
      └─ oht_data_m16BR.csv ─────┘── VelocityTracker ───── (B) 차량 순간속도
          VEHICLE → vid                   ↑                   m/min
```

---

OHS 데이터에서 차량 속도와 관련된 값은 **세 가지**가 있고 의미가 모두 다르다.
혼동을 피하기 위해 먼저 구분한다.

| 구분 | 값 | 단위 | 의미 | 데이터 소스 |
|---|---|---|---|---|
| **(A) ZONE 자유흐름속도** | `FREE_FLOW_SPEED` | m/min | **HID 구간(=ZONE) 자체의 nominal 속도**. 차량이 누구든 그 구간 통과 시 거의 동일 값으로 기록됨 | `LOGPRESSO_HID_INOUT_*.csv` |
| **(B) OHT 순간 속도** | VelocityTracker 출력 | m/min | 차량별 **0.5–1초 해상도** UDP 위치차분 속도 | `LOGPRESSO_oht_data_*.csv` |
| **(C) OHT 구간 평균 속도** | HID 통과시간 차분 | m/min | 같은 차량의 두 HID 통과 사이 평균 속도 (보조 지표) | `LOGPRESSO_HID_INOUT_*.csv` |

> **핵심**: `FREE_FLOW_SPEED` 는 **차량별 속도가 아니라 ZONE 별 속도**다. 같은 HID(예: 19→13) 를 통과한 차량 30대의 FFS 가 모두 152~154 m/min 좁은 범위에 모인다(샘플 검증 완료). 차량별 실제 속도는 **반드시 (B) 또는 (C) 로 따로 계산**해야 한다.

## 1. (B) OHT 순간 속도 — VelocityTracker (권장)

### 1-1. 입력 데이터

`LOGPRESSO_oht_data_<fab>_<date>.csv` (M16A/BR 의 경우 `LOGPRESSO_oht_data_m16BR_20260421.csv`) 의 컬럼 매핑:

| velocity_tracker 인자 | CSV 컬럼 | 단위 | 비고 |
|---|---|---|---|
| `vid` | `VEHICLE` | str | 예: `BV0077` |
| `t` | `_time` | datetime | UDP 수신 시각 |
| `curr_node` | `ADDRESS` | int | 현재 세그먼트 시작 노드 |
| `next_node` | `NEXT_ADDRESS` | int | 현재 세그먼트 끝 노드 |
| `distance` | `DISTANCE` | **dm** (=100mm) | `ADDRESS` 로부터 진행 거리 |
| `edge_dist_map[(a,b)]` | `layout_cache.json` | **dm** | `a→b` 엣지 전체 길이 |

> 거리 단위는 UDP/layout 모두 **dm (1dm=100mm)** 로 통일 — 변환 상수 없이 산술 가능.

---

### 1-2. 기본 공식

```
speed [m/min] = Δdistance_dm × 6 / Δt_sec
```

#### 유도
- 1 dm = 0.1 m
- 1 분 = 60 초
- `(Δdm × 0.1 m/dm) / Δt_sec × 60 s/min = Δdm × 6 / Δt_sec  [m/min]`

즉 **계수 6 = 0.1 × 60**. (`velocity_tracker.py:115`)

#### 단위 변환표 (m/min ↔ km/h)

| m/min | km/h | 비고 |
|---:|---:|---|
| 60   | 3.6  | 사람 빠른 걸음 |
| 100  | 6.0  | OHT 저속 |
| 180  | 10.8 | DEFAULT_VELOCITY (기본 fallback) |
| 250  | 15.0 | 일반 운행 속도대 |
| 300  | 18.0 | MAX_SPEED (안전 상한, 클램프) |

> 변환식: `km/h = m/min × 60 / 1000 = m/min × 0.06`

---

### 1-3. 세 가지 계산 케이스

`prev.edge = (prev_curr, prev_next)`, 현재 `edge = (curr, next)` 로 두고 `Δdm` 계산 방식은 세 가지:

### Case A — 같은 엣지 내부 이동 (`prev.edge == edge`)
```
Δdm = distance - prev.distance
```
- `velocity_tracker.py:95-97`
- 가장 일반적인 경우. 같은 구간을 연속 통과 중.

### Case B — 자연스러운 엣지 전환 (`prev.edge[1] == edge[0]`)
```
prev_len = edge_dist_map[prev.edge]
Δdm     = (prev_len - prev.distance) + distance
```
- `velocity_tracker.py:98-101`
- 이전 엣지를 마저 끝내고 새 엣지로 넘어간 총 이동량.

### Case C — 불연속(샘플 누락) — BFS 경로 추정
```
path_len = BFS 최단경로 길이 (prev.edge[1] → edge[0])
prev_len = edge_dist_map[prev.edge]
Δdm     = (prev_len - prev.distance) + path_len + distance
```
- `velocity_tracker.py:103-110`
- UDP 드롭이나 프레임 점프로 엣지가 떨어진 경우.
- BFS 탐색 한도는 `BFS_MAX_HOPS = 20`. 경로 못 찾으면 측정 불가 처리.
- 결과는 `_bfs_cache` 에 저장해 같은 노드쌍 재조회 가속.

---

### 1-4. 안전 가드 / 필터링

| 조건 | 처리 | 위치 |
|---|---|---|
| 최초 관측 (`prev is None`) | `None` 반환, prev 저장 | 73-75 |
| 같은 시각 재호출 (`t == prev.time`) | 마지막 계산값 캐시 반환 | 77-79 |
| 과거 UDP 늦게 도착 (`t < prev.time`) | 캐시 반환, prev 갱신 X | 81-83 |
| `elapsed <= 0` | 캐시 반환 | 86-88 |
| `elapsed > MAX_GAP_SEC (30s)` | 측정 불가, prev 갱신 후 캐시 반환 | 91-93 |
| `Δdm < 0` (후진 / distance 리셋) | `speed = 0.0` | 112-113 |
| `NaN` / `inf` | 측정 불가, prev 갱신 후 캐시 반환 | 116-118 |
| 범위 벗어남 | `max(MIN_SPEED, min(MAX_SPEED, speed))` 클램프 | 119 |

### 상수
```python
MIN_SPEED    = 0.0     # m/min  (정지 허용; ATLAS는 1.5)
MAX_SPEED    = 300.0   # m/min  (= 18 km/h, 안전 상한)
MAX_GAP_SEC  = 30.0    # 이 이상 갭은 평균속도로 의미 없음
BFS_MAX_HOPS = 20
```
(`velocity_tracker.py:40-43`)

### 1-5. 차량별 상태 유지

`VelocityTracker` 는 차량 ID(`vid`) 별로 두 개의 맵을 보관:

- `self._prev[vid]: _PrevState(time, edge, distance)` — 마지막 UDP 스냅샷
- `self._last_speed[vid]: float` — 마지막 계산 속도(캐시)

`reset(vid)` 로 개별 차량만, `reset()` 로 전체 초기화 가능. 프레임 점프 / 시뮬레이션 리플레이 점프 시 호출.

### 1-6. 호출 지점 (연동)

`WORD_MODEL/WORLD_SIM/world_model.py:191-224` 의 `load_vehicles_from_frame()` 에서 매 프레임 호출:

```python
self.velocity_tracker = VelocityTracker(self.edge_dist_map, adj_map)
```

- `None` 을 반환하면 UI 는 "측정 불가" 로 표기 (기본 속도로 메우지 않음).

## 2. (A) ZONE 자유흐름속도 — `FREE_FLOW_SPEED`

`LOGPRESSO_HID_INOUT_*.csv` 의 `FREE_FLOW_SPEED` 컬럼은 **HID 구간(=ZONE) 자체의 자유흐름속도** 다. 차량마다 미세하게 다른 값이 적힌 것은 측정 노이즈일 뿐, **본질적으로 ZONE 의 속성**.

### 2-1. 컬럼 구조

| 컬럼 | 의미 |
|---|---|
| `_time` | HID 통과(=구간 진입/이탈) 시각 |
| `VHL_ID` | 통과한 차량 ID |
| `FROM_HIDID` → `TO_HIDID` | HID 진입/이탈 노드 |
| `FREE_FLOW_SPEED` | **그 ZONE 의 자유흐름속도** (m/min) |
| `MCP_NM` | MCP (M16A 의 경우 `BR`) |
| `VHL_COUNT_LIMIT` | 그 ZONE 의 차량 수 제한 |
| `VHL_PRECAUTION` | 주의 표시 |

### 2-2. 검증 (M16A_BR 20260421 데이터)

같은 HID 인 `FROM=19, TO=13` 을 통과한 30대 차량의 FFS:

```
152.48, 152.38, 152.46, 152.39, 152.26, 151.98, 153.46, 153.64, ...
→ 평균 약 153.2 m/min, 표준편차 ≈ 0.7 m/min
```

→ **차량 무관, ZONE 별로 거의 일정**. 따라서 FFS 는 OHT 의 실제 속도가 **아니다**.

### 2-3. 활용

- ZONE 별 속도 한계 / 노미널 속도 참조용 (시각화, 검증)
- `data_loader.py:625-657` `_load_hid_inout()` 가 `speed = FREE_FLOW_SPEED` 로 읽어 `get_hid_speed_summary()` 에서 ZONE 별 평균/최저/최고로 집계

## 3. (C) HID 통과 시간 차분 — OHT 평균 속도 (보조)

OHT raw 가 없을 때, **`LOGPRESSO_HID_INOUT_*.csv` 만으로도** 차량별 평균 속도를 도출할 수 있다.

### 3-1. 원리

같은 `VHL_ID` 의 두 연속 HID 통과 사이:

```
v_avg [m/min] = path_length(HID_n → HID_{n+1}) / (t_{n+1} − t_n)
```

- `path_length` = layout_cache 에서 두 HID 사이 엣지 길이 합 (dm) → m 변환
- `t` 차이 = 분 단위로 환산
- 차량별로 시계열 정렬 후 차분

### 3-2. 데이터 검증 (BV0103 예시)

```
14:00:00.873  HID 19→13   FFS=152.48
14:00:00.873  HID 21→20   FFS=123.70
14:00:00.874  HID 20→14   FFS=139.92
14:00:00.874  HID 14→19   FFS=126.63
14:01:00.431  HID 13→6    FFS=121.60
14:03:00.213  HID  6→2    FFS=124.90
14:04:00.283  HID  2→3    FFS=128.52
```

차량 BV0103 의 14:00:00.874 → 14:01:00.431 사이(약 60s) HID 14→19 다음 13→6 까지의 경로 길이를 알면, 60초 동안의 평균 속도가 나온다.

### 3-3. 한계

- **해상도가 낮다**: HID 통과 간격이 수십 초~수 분 단위라 순간 속도가 아닌 평균 속도만 가능
- 경로 길이 계산이 BFS/그래프 의존 → 분기/우회 시 실제 경로와 차이
- 정밀한 시뮬레이션/대시보드용은 (B) VelocityTracker 권장
- 단, HID_INOUT 만 가용한 환경(e.g. 간이 분석)에서는 유용

|---|---|---|---|
| 대상 | ZONE | 차량 | 차량 |

## 5. 예시 계산 (B - VelocityTracker)

### 예 1 — Case A (같은 엣지)
- `prev`: t=14:00:00.000, edge=(19,2), distance=100
- `now `: t=14:00:01.000, edge=(19,2), distance=250
- `Δt = 1.0s`, `Δdm = 150`
- **speed = 150 × 6 / 1.0 = 900 m/min** → 클램프 → **300 m/min (MAX)**

### 예 2 — Case B (엣지 전환)
- `prev`: t=14:00:00.000, edge=(19,2), distance=350 (엣지 길이 400 dm)
- `now `: t=14:00:01.000, edge=(2,5), distance=80
- `Δt = 1.0s`, `Δdm = (400-350) + 80 = 130`
- **speed = 130 × 6 / 1.0 = 780** → **300 m/min (MAX)**

### 예 3 — 정상 범위
- `Δdm = 30`, `Δt = 1.0s`
- **speed = 30 × 6 / 1.0 = 180 m/min** (≈10.8 km/h)

### STEP 1 · `LOGPRESSO_HID_INOUT.csv` 읽기 → (A) ZONE 속도

**파일**: `M16A_BR_DATA/LOGPRESSO_HID_INOUT_20260421.csv`
**사용 컬럼**: `FROM_HIDID`, `TO_HIDID`, `FREE_FLOW_SPEED`, `VHL_ID`, `_time`

```python
import csv
from datetime import datetime

with open("LOGPRESSO_HID_INOUT_20260421.csv", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    same_zone_ffs = []     # (A) 검증용
    bv0103_passes = []     # 차량 시계열
    for row in reader:
        fr  = int(row["FROM_HIDID"])
        to  = int(row["TO_HIDID"])
        ffs = float(row["FREE_FLOW_SPEED"])
        if fr == 19 and to == 13:           # 같은 ZONE 통과 모음
            same_zone_ffs.append(ffs)
        if row["VHL_ID"] == "BV0103":
            bv0103_passes.append((row["_time"], fr, to, ffs))

# 결과: same_zone_ffs 의 평균 ≈ 154.33, std ≈ 1.46  → ZONE 속성 확인
```

**기대 결과** — 같은 HID(19→13) 통과 519건의 FFS 가 좁은 범위:
```
평균 154.33 m/min   표준편차 1.46 m/min
→ 차량 무관, ZONE 속성
```

### STEP 2 · `layout_cache.json` 로드 → VelocityTracker 입력

**파일**: `WORD_MODEL/OHT_MAP/cache/M16A_BR_layout_cache.json`
**사용 키**: `edges` (구간 길이), `adj` (인접 노드)

```python
import json

with open("M16A_BR_layout_cache.json", encoding="utf-8") as f:
    d = json.load(f)

adj_map  = {int(k): list(v) for k, v in d["adj"].items()}
```

**구조 예시**:
```
edges: "1,101"   → 360       # dm
```

### STEP 3 · OHT raw 읽기 + (B) `VelocityTracker` 로 순간속도 계산

**파일**: `M16A_BR_DATA/20260421.zip` 안의 `LOGPRESSO_oht_data_m16BR_20260421.csv`
**컬럼 매핑**:

| CSV | → | tracker 인자 |
|---|---|---|
| `VEHICLE` | → | `vid` |
| `_time` | → | `t` (datetime) |
| `ADDRESS` | → | `curr_node` |
| `NEXT_ADDRESS` | → | `next_node` |
| `DISTANCE` | → | `distance` (dm) |

```python
import csv, zipfile
from datetime import datetime
from velocity_tracker import VelocityTracker  # WORD_MODEL/WORLD_SIM/

tracker = VelocityTracker(edge_dist_map=edge_map, adj_map=adj_map)

# zip 안 CSV 스트리밍 읽기 (메모리 절약)
with zipfile.ZipFile("20260421.zip") as zf, \
     zf.open("LOGPRESSO_oht_data_m16BR_20260421.csv") as raw:
    text = (line.decode("utf-8", errors="replace") for line in raw)
    rows = []
    for row in csv.DictReader(text):
        if row["VEHICLE"] != "BV0103":
            continue
        t = datetime.strptime(row["_time"].strip(), "%Y-%m-%d %H:%M:%S.%f%z").replace(tzinfo=None)
        rows.append((t, int(row["ADDRESS"]), int(row["NEXT_ADDRESS"]), int(row["DISTANCE"])))

rows.sort()  # ★ 시간순 정렬은 필수

# 한 줄씩 update() 호출 → 트래커 내부에서 Δdm × 6 / Δt 자동 계산
for t, addr, nxt, dist in rows:
    speed = tracker.update("BV0103", t, addr, nxt, dist)
    if speed is not None:
        print(f"{t}  {addr}→{nxt}  {speed:.2f} m/min ({speed*0.06:.2f} km/h)")
```

**해석 가이드**:
- `speed = None` → 첫 관측 / 갭 30s 초과 / 후진 / NaN
- `speed = 0` → 정지 (또는 후진 보정)
- `speed = 300` → MAX 클램프 (Δt 매우 짧고 Δdm 큼)
- 정상 분포는 **100~250 m/min (6~15 km/h)**

### STEP 4 · HID 통과 차분 → (C) OHT 평균속도 (보조)

OHT raw 가 없을 때만 사용. 같은 차량의 두 연속 HID 통과 시각 차이로 평균속도 산출.

```python
import csv
from datetime import datetime

rows = []
with open("LOGPRESSO_HID_INOUT_20260421.csv", encoding="utf-8") as f:
    for row in csv.DictReader(f):
        if row["VHL_ID"] != "BV0103":
            continue
        t = datetime.strptime(row["_time"].strip(), "%Y-%m-%d %H:%M:%S.%f%z").replace(tzinfo=None)
        rows.append((t, int(row["FROM_HIDID"]), int(row["TO_HIDID"])))
rows.sort()

PATH_M = 250.0  # ZONE 당 경로 길이 (실제는 layout 으로 계산)
for i in range(1, len(rows)):
    dt = (rows[i][0] - rows[i-1][0]).total_seconds()
    if dt < 1.0:           # 동시각 다중 HID 행 스킵
        continue
    v_avg = PATH_M / dt * 60  # m/min
    print(f"{rows[i][1]}→{rows[i][2]}  Δt={dt:.1f}s  v_avg={v_avg:.2f} m/min ({v_avg*0.06:.2f} km/h)")
```

**예상 출력 (BV0103)**:
```
13→6   Δt=59.56s  v_avg=251.86 m/min (15.11 km/h)
 6→2   Δt=119.78s v_avg=125.23 m/min ( 7.51 km/h)
 2→3   Δt=60.07s  v_avg=249.71 m/min (14.98 km/h)
```

**한계**: 해상도 낮음(수십초~수분), 경로 추정에 의존 — UI 가 아니라 분석 보조용.

## 8. 결론

1. **`FREE_FLOW_SPEED` 는 ZONE 별 자유흐름속도** — OHT 의 실제 속도가 아님 (검증 완료: 같은 HID 30대 차량 FFS 표준편차 ≈ 0.7 m/min)
2. **OHT 개별 순간 속도 = (B) VelocityTracker** (`velocity_tracker.py:115`)
   - 공식: `speed = Δdm × 6 / Δt_sec`
   - 입력: `LOGPRESSO_oht_data_<fab>_<date>.csv` (UDP raw)
   - 컬럼: `VEHICLE`, `_time`, `ADDRESS`, `NEXT_ADDRESS`, `DISTANCE`
   - 케이스: 같은 엣지(A) / 자연 전환(B) / BFS 보간(C)
   - 가드: MIN=0 / MAX=300 / MAX_GAP=30s / 후진 0 / NaN 필터
3. **OHT 평균 속도 (보조) = (C) HID 통과 차분** — HID_INOUT 만 가용 시 사용
4. 시뮬레이터 UI 의 차량 속도는 항상 (B) 사용. ZONE 통계는 (A), 분석 보조는 (C).
