## 5. 시사점

1. **예측 모델 시간 축**
   - 월드모델/데드락 예측에서 OHT 상태 변화는 **≥1.7초 해상도**로만 관측 가능. 서브초(sub-second) 이벤트는 소실됨 (< 0.5초 간격은 전체의 6.6%).
2. **0.3초 주기 전제 금지**
   - 만약 UDP 단말 레벨에서 0.3초 송신을 수행하더라도 Logpresso 적재 단계에서 sampling/중복제거로 약 1/5~1/6 수준으로 감소.
   - 고정밀 제어·충돌 검증용이라면 raw UDP 캡처(별도 파이프라인) 필요.
3. **시스템 부하**
   - 393 msg/s × 평균 라인 크기 ≈ **50 KB/s** 수집 대역 (M14A만).
   - 하루 약 3,400만 건 → 월 10억 건 수준 스토리지 설계.
4. **오래 끊기는 차량(>5초)**
   - 3.7% 구간: 대기/충전/오프라인 추정. 월드모델에서 "실종" 차량 처리 정책 필요.

## 6. 재현 방법

```bash
cd WORD_MODEL/OHS_DATA_MD/20260421/
cat 20260421.zip.0?? > merged.zip
unzip merged.zip
python3 run_encoder.py 20260421 .
unzip 20260421.zip -d ./out/
# 주기 분석
python3 <<'PY'
import csv, statistics
from collections import defaultdict
from datetime import datetime
vt = defaultdict(list)
with open('out/LOGPRESSO_OHT_DATA_20260421.csv') as f:
    r = csv.reader(f); h = next(r); ti = h.index('_time'); li = h.index('line')
    for i, row in enumerate(r):
        if i > 2_000_000: break
        try: t = datetime.strptime(row[ti][:23], "%Y-%m-%d %H:%M:%S.%f")
        except: continue
        v = row[li].split(',')[2]
        vt[v].append(t)
d = []
for ts in vt.values():
    ts.sort()
    d += [(ts[i+1]-ts[i]).total_seconds() for i in range(len(ts)-1)]
print(f"median={statistics.median(d):.3f}s mean={statistics.mean(d):.3f}s n={len(d)}")
PY
```

기대 출력: `median=1.700s mean=2.702s n=1998937`
