# 3초 주기 외부 송출 설계 — OHT 속도 & HID ZONE 속도

> 작성일: 2026-04-23
> 대상 독자: 고객 / 연동 상대 시스템 담당자
> 관련 문서: [OHT_VHL_속도_계산_20260422.md](./OHT_VHL_속도_계산_20260422.md)

---

## 1. 개요

현재 OHS 시스템은 **UDP 를 고해상도(median 1.6초)** 로 수신해 내부에서 차량별/구간별 속도를 계산한다. 이제 외부 시스템으로 **3초에 한 번** 속도 데이터를 송출하는 요구가 추가됐다.

> 내부 수신은 그대로 두고, **외부 송출만 3초로 다운샘플**한다.

송출 대상 두 가지:
1. **OHT 속도** — 차량(VHL) 별 속도 (VelocityTracker 계산값)
2. **HID ZONE 속도** — 구간(HID) 별 속도 (FREE_FLOW_SPEED)

각 3초 윈도우에서 **평균** 을 대표값으로 하고, `min`, `max`, `n(샘플 수)` 를 참고 지표로 함께 보낸다.

---

## 2. 배경 — 현재 내부 시스템

| 항목 | 값 | 비고 |
|---|---|---|
| UDP 수신 주기 (median) | 1.6 초 | BV0103 실측, 20260421 데이터 |
| OHT 속도 계산 | `VelocityTracker.update()` | `speed [m/min] = Δdm × 6 / Δt_sec` |
| ZONE 속도 필드 | `FREE_FLOW_SPEED` | HID_INOUT CSV, m/min |
| ZONE 속도의 성격 | ZONE 속성 | 같은 HID 30대 표준편차 ≈ 1.5 → 차량 무관 |
| 안전 상한 | 300 m/min (= 18 km/h) | `MAX_SPEED` 클램프 |

자세한 공식·단위·케이스 분기는 [OHT_VHL_속도_계산_20260422.md](./OHT_VHL_속도_계산_20260422.md) 참조.

## 3. 데이터 흐름

```
[내부 (고해상도 유지)]                                 [외부 송출 (신규, 3s)]
────────────────────────────────────                   ─────────────────────────────────
 UDP ──► VelocityTracker ──► per-vehicle speed ──┐
                                                 │
                                                 ├─► 3초 tumbling 버퍼 ──► 평균(+min/max/n)
                                                 │
 HID_INOUT ──► FFS 기록 ────► per-zone speed ────┘                │
                                                                   │
                                                                   ▼
                                                        ┌──────────────────────┐
                                                        │  payload (3초마다)   │
                                                        │  t_start, t_end      │
                                                        │  vhl_speed[vid]={..} │
                                                        │  zone_speed[k]={..}  │
                                                        └──────────────────────┘
```

- **Tumbling 윈도우**: `[t, t+3초)`, `[t+3, t+6)`, … 중첩 없음. 각 UDP 샘플은 정확히 한 윈도우에만 귀속.
- **Pull 모델**: 외부 송출 컴포넌트가 3초마다 버퍼 flush → 전송.

## 4. 집계 정책

### 4-1. 대표값은 **평균(arithmetic mean)**

한 차량(또는 한 ZONE) 이 3초 구간 안에서 기록한 속도 샘플들의 산술평균.

| 이유 | 설명 |
|---|---|
| 실제 평균 거동 반영 | 3초 동안의 일반적인 속도를 그대로 나타냄 |
| 이상치 강건 | 단일 스파이크가 대표값을 휘두르지 않음 |
| 관제 표준 | 속도·유량 계열 지표는 평균이 일반적 |

### 4-2. 다른 집계 방식과의 비교

| 방식 | 장점 | 단점 | 채택 |
|---|---|---|---|
| **평균** | 실제 거동 대표, 이상치 강건 | 스파이크 손실 | ✅ 메인 |
| MAX | 상한/이상 감지 | 노이즈 1건에 민감 | 보조 (`max` 필드) |
| MIN | 정체/저속 감지 | 정지/후진 0 영향 | 보조 (`min` 필드) |
| LATEST | 최신성 | 대표성 부족 | ❌ |
| EMA | 부드러움 | 지연 발생, 윈도우 경계 모호 | ❌ |

> 수신측에서 `max` / `min` / `n` 을 보내주므로, 필요 시 그 쪽에서 재해석 가능.

### 4-3. 결측/특수값 처리
- `VelocityTracker` 가 `None` 반환한 샘플은 버퍼에 담지 않음 (`n` 카운트 X)
- 3초간 샘플 0건인 차량/ZONE 은 payload 에서 **키 자체를 생략** (빈 객체 방지)
- 시간 점프/리플레이 리셋 시 버퍼 전체 flush

## 5. 송출 포맷 (JSON)

### 5-1. 스키마

```json
{
  "t_start": "2026-04-21T14:00:00",
  "t_end":   "2026-04-21T14:00:03",
  "vhl_speed": {
    "<VHL_ID>": {"avg": <float>, "min": <float>, "max": <float>, "n": <int>}
  },
  "zone_speed": {
    "<FROM_HIDID>_<TO_HIDID>": {"avg": <float>, "min": <float>, "max": <float>, "n": <int>}
  }
}
```

### 5-2. 실제 예시

```json
{
  "t_start": "2026-04-21T14:00:00",
  "t_end":   "2026-04-21T14:00:03",
  "vhl_speed": {
    "BV0103": {"avg": 240.45, "min": 232.10, "max": 248.76, "n": 2},
    "BV0085": {"avg": 198.43, "min": 187.10, "max": 201.33, "n": 2}
  },
  "zone_speed": {
    "19_13":  {"avg": 153.20, "min": 152.50, "max": 154.10, "n": 3},
    "20_14":  {"avg": 144.50, "min": 141.30, "max": 147.22, "n": 2}
  }
}
```

### 5-3. 필드 정의

| 필드 | 타입 | 단위 | 의미 |
|---|---|---|---|
| `t_start` | string (ISO8601) | — | 윈도우 시작 시각 (포함) |
| `t_end` | string (ISO8601) | — | 윈도우 끝 시각 (불포함) |
| `vhl_speed[VHL_ID].avg` | float | m/min | 차량 3초 평균속도 |
| `vhl_speed[VHL_ID].min` | float | m/min | 최솟값 |
| `vhl_speed[VHL_ID].max` | float | m/min | 최댓값 |
| `vhl_speed[VHL_ID].n`   | int   | — | 샘플 수 |
| `zone_speed["F_T"].avg` | float | m/min | ZONE(F→T) 3초 평균 FFS |
| `zone_speed["F_T"].min` | float | m/min | 최솟값 |
| `zone_speed["F_T"].max` | float | m/min | 최댓값 |
| `zone_speed["F_T"].n`   | int   | — | 샘플 수 |

### 5-4. 단위 변환

| m/min | km/h | m/s |
|---:|---:|---:|
| 60  | 3.6  | 1.00 |
| 180 | 10.8 | 3.00 |
| 240 | 14.4 | 4.00 |
| 300 | 18.0 | 5.00 |

변환식: `km/h = m/min × 0.06`, `m/s = m/min / 60`.

## 6. 실데이터 예시 (M16A_BR 20260421)

### 6-1. OHT 속도 — 차량 `BV0103`, 14:00:00 ~ 14:00:03 윈도우

내부 VelocityTracker 출력 가정 시계열:
```
14:00:01.075  → None  (첫 관측)
14:00:02.302  → 232.10 m/min
14:00:02.992  → 248.76 m/min
```

송출 payload:
```json
"BV0103": {"avg": 240.43, "min": 232.10, "max": 248.76, "n": 2}
```
- `n = 2` (None 샘플 제외)
- 평균 (232.10 + 248.76) / 2 = 240.43

### 6-2. HID ZONE 속도 — `19→13`, 같은 3초 윈도우

HID_INOUT 이벤트로 들어온 FFS 값:
```
14:00:00.873  → 152.48
14:00:00.874  → 153.00
14:00:02.500  → 154.10
```

송출 payload:
```json
"19_13": {"avg": 153.19, "min": 152.48, "max": 154.10, "n": 3}
```

### 6-3. 한 payload 종합

```json
{
  "t_start": "2026-04-21T14:00:00",
  "t_end":   "2026-04-21T14:00:03",
  "vhl_speed":  { "BV0103": {"avg": 240.43, "min": 232.10, "max": 248.76, "n": 2} },
  "zone_speed": { "19_13":  {"avg": 153.19, "min": 152.48, "max": 154.10, "n": 3} }
}
```

## 7. 엣지 케이스

| 상황 | 처리 |
|---|---|
| 3초 내 샘플 0건 | 해당 `vhl_speed[vid]` / `zone_speed[key]` **키 생략** |
| 3초 내 샘플 1건 | `avg = min = max`, `n = 1` |
| `VelocityTracker` None | 버퍼에 안 담음 (n 에 미포함) |
| 시간 점프(리플레이 시점 변경) | 버퍼 전체 리셋, 다음 3초부터 재시작 |
| 윈도우 경계 딱 걸치는 샘플 | `t ∈ [t_start, t_end)` 규칙 (좌폐우개) |
| 이상 속도 (계산상) | VelocityTracker 가 0~300 으로 클램프해서 넣음 |

## 10. 검증 (Acceptance Criteria)

상대 시스템과 합의할 수용 기준:

1. **주기 정확도**: 두 payload 간격 = 3.0 초 ± 100 ms
2. **값 불변식**: 모든 레코드에 대해 `min ≤ avg ≤ max`, `n ≥ 1`
3. **평균 일치**: 같은 3초 구간 내부 고해상도 샘플의 산술평균과 payload `avg` 일치 (부동소수 오차 허용)
4. **None 제외**: `VelocityTracker` 가 `None` 반환한 샘플은 `n` 에 포함되지 않음
5. **내부 영향 없음**: 기존 대시보드/시뮬레이터 UI 는 그대로 작동 (외부 송출은 별도 라인)
6. **단위 일치**: 모든 속도 값이 m/min (`MAX_SPEED=300` 을 초과하지 않음)
7. **타임존 일치**: `t_start`, `t_end` 의 타임존 표기 방식이 합의된 기본안과 일치

## 12. 요약

- **내부**: UDP 전부 수신, VelocityTracker 고해상도 유지 (변경 없음)
