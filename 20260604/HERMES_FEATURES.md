## 📅 어제 만든 것 (2026-06-04) — 코어 기능 6종

엔진·저장계층·프로토콜·UI·납품본까지 헤르메스의 뼈대 전부.

| # | 기능 | 발동/시점 | 하는 일 |
|---|---|---|---|
| A1 | 🧠 **장기 메모리** | 사용자가 역할·환경·선호를 드러낼 때 | MEMORY(사실)/USER(선호) 자동 저장 → 새 세션·기기에도 유지 |
| A2 | 📚 **자기학습 스킬** | 재사용 가치 있는 절차 발견 시 | SKILL.md로 생성/patch (**확인형** — 승인 후 저장) + BM25 회상 |
| A3 | ❓ **되묻기 (clarify)** | 요청이 모호할 때 | 추측 말고 핵심 1~3개 먼저 질문 |
| A4 | 🔎 **세션 회상** | 🔮 → 대화검색 탭 / 필요 시 | 과거 대화 BM25 검색 (LLM 없이) |
| A5 | ♻️ **백그라운드 리뷰** | 10턴마다 자동 | 놓친 기억을 백그라운드 API로 보충 (본 대화 안 막음) |
| A6 | 🗂 **큐레이터** | 앱 켤 때 1회 | 30일 미사용 스킬→stale, 90일→archived (pinned 면제) |

**같이 만든 것**: 텍스트 프로토콜 파서, 파일 저장계층(원자쓰기+인젝션 차단), Flask 라우트(`/api/hermes/*`), 기억/스킬/검색 뷰어 UI, MEMORY 자동저장 강화, 독립 납품본(`agent-kits/hermes-engine/`).

## 📅 오늘 만든 것 (2026-06-05) — 빌트인 스킬 팩 4종

Hermes(NousResearch)에서 가져온, 폐쇄망에서 안전한 **프롬프트 스킬**. 질문 키워드에 맞으면 **자동 주입**. (execute_code/web 등 위험·외부망은 제외)

| # | 스킬명 | 발동 키워드 | 하는 일 |
|---|---|---|---|
| B1 | 🧭 **task-planning** | 만들·구현·설계·프로그램·코드·연구·분석·계획·파이프라인·단계·아키텍처 | 다단계 작업 → 목표정의 → 3~6단계 계획 → 실행 → 자가점검 |
| B2 | 🐞 **systematic-debugging** | 오류·에러·버그·디버·안돼·실패·exception·traceback·고장 | 재현 → 가설2~3 → 최소수정 검증 → 환경vs코드 구분 → 재검증 |
| B3 | 📊 **structured-data-analysis** | 데이터·분석·통계·반송·FAB·로그·수율·OHT·OHS·CSV·컬럼·추세 | 컬럼/단위 확인 → 가설 → 통계→이상치→추세 → 실제값만 → 시각화 |
| B4 | ✅ **answer-verification** | 보고서·결론·정확·검증·요약 | 최종답 전 자가검증: 근거 실재? 누락? 불확실하면 명시 |

## ⚙️ 작동 방식 (공통)

- **텍스트 프로토콜**: 모델이 답변에 펜스 블록을 출력하면 백엔드가 파싱·처리 (네이티브 함수콜 불필요)
  - ` ```hermes:memory ` (기억 저장) · ` ```hermes:skill ` (스킬 생성, 확인형) · ` ```hermes:ask ` (되묻기)
- **파일 저장**: `demos_data/agents/<user_id>/`
  - `MEMORY.md` / `USER.md` (§ 구분) · `skills/<이름>/SKILL.md` · `sessions/YYYY-MM-DD.jsonl` · `usage.json` · `counters.json` · `curator.json`
  - 모든 쓰기 `tempfile → os.replace` 원자 교체, 인젝션 패턴 차단, 저장소당 6,000자 상한
