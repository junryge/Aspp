🛠 Harness MVP 도구 실행 하네스

Python 도구 실행 하네스 — 도구를 등록하고 · 프롬프트로 알맞은 도구를 골라 · 권한 체크하며 · 멀티턴으로 실행/스트리밍하고 · 세션·기록을 저장하고 · 피드백으로 개선한다.

📦 모듈별 기능

모듈기능핵심 API
registry 도구 등록 / 관리 / 실행 register unregister get list_all find filter execute
router 프롬프트 → 도구 매칭 route — 토큰 스코어링으로 알맞은 도구 선택
permissions 권한 차단(deny-list) + 코드 안전검사 ToolPermissionContext.blocks, CodeGuard.check / is_safe — 위험 패턴 차단
engine 실행 엔진 (핵심) submit(1턴) run_loop(멀티턴) stream_submit(스트리밍) · 예산관리 · persist(세션 저장) · from_session(복원)
session 세션 저장 / 로드 StoredSession — JSON 영속
history 이벤트 로그 add, as_markdown — 실행 기록
transcript 트랜스크립트 관리 append / compact(압축) / replay(재생) / flush
context 워크스페이스 스캔 WorkspaceContext, ProjectMap — 프로젝트 파일 맥락 파악
expert_pool 에이전트 동적 선택 질문에 맞는 전문가 에이전트만 골라 배정 (AgentAssignment)
feedback 피드백 루프 (학습) add(👍👎) / get_summary / build_prompt_hint / add_mistake / get_mistakes — 실수 회피
models 핵심 데이터 구조 Tool / Task / ToolExecution / TurnResult / UsageSummary 등
cli 커맨드라인 python -m harness list / find / route

🔄 실행 흐름

도구 등록 → 프롬프트 라우팅 → 권한 체크 → 멀티턴 실행 / 스트리밍 → 세션 · 기록 저장 → 피드백으로 개선
registry → router → permissions → engine(run_loop/stream) → session/history/transcript → feedback

⚖️ 하네스 vs 헤르메스

🛠 하네스 (harness-mvp)

“무엇을 한다” — 도구 실행
  • 도구 등록·라우팅·실행
  • 권한·멀티턴·세션
  • 피드백 학습 · 전문가 선택

🔮 헤르메스 (hermes-engine)

“무엇을 안다” — 기억·학습
  • 사용자 사실/선호 기억
  • 자기학습 스킬
  • 되묻기 · 세션 회상

💡 둘은 독립적이며, 합치면 “기억하고 도구도 쓰는 에이전트” 가 됩니다 (통합은 개발자 몫).

cd harness-mvp
