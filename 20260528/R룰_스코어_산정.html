0. 전체 스코어 구조 (한눈에)

3단계 합산 구조
  1. 룰 트리거 평가 — 각 룰 (R-A, R-B, R-C, R-D + SLA, Sorter, MAXCAPA) 의 발동 여부 결정
  2. 영역별 점수 합산 (0~50) — 8 영역마다 발동된 룰 점수 누적
  3. 통합 점수 계산 (0~500) — 영역 점수 + 흐름 + SLA + Sorter + MAXCAPA + 추가 가중치
단계 대상 가산 최대
1단계 룰 트리거 판정 True / False (구간/임계 조건) —
2단계 영역 점수 (8영역 각각) RA+10 / RA_sus+5 / RB+10 / RB_fast+5 / RC+8 / RD+7 / SLA+5 / SORT+3 / MAXCAPA+10n 50/영역
3단계 통합 점수 Σ 영역점수 + flow(+5/+15/+30) + sla(+5) + sorter(+3) + maxcapa(+10n) 500

1. R-A (반송시간 정체)

반송시간이 임계 이상으로 길어진 사건

측정 컬럼 (영역별 다름):

영역컬럼임계 (분)

M14BM14B.QUE.TIME.AVGTOTALTIME1MIN5.0

1-1. RA (절대 임계) — +10점

조건: 최근 10분 윈도우 중 1회 이상 값 ≥ TH_RA[area]
예: M16HUB.AVGTOTALTIME1MIN 가 10분 중 11.2분이 1번이라도 찍히면 → RA trigger ON → 영역 점수 +10

1-2. RA_sus (지속 임계) — +5점

조건: 최근 5분 중 3회 이상 값 ≥ (TH_RA × 0.67)
상수: TH_RA_SUSTAINED_RATIO = 0.67, TH_RA_SUSTAINED_COUNT = 3
예: M16HUB 의 sustained 임계 = 9.0 × 0.67 = 6.03 분. 5분 중 3분이 ≥ 6.03 → RA_sus ON → +5
왜 두 임계? RA 는 "갑작스런 spike", RA_sus 는 "만성 정체". 9분 한 번보다 6분이 3번이 더 무서움. 두 트리거가 동시 발동하면 +10 + +5 = +15 가산.

2. R-B (큐 누적)

큐가 시간에 따라 쌓이는 사건 (사용자 예시: M14TOM16 +100)

측정 컬럼:

영역컬럼30분 임계 (+N)10분 임계 (+M)
M16HUBM16HUB.QUE.M14TOM16.MESCURRENTQCNT10030
M14M14.QUE.ALL.3F_TO_HUB_JOB8024
M14BM14B.QUE.ALL.7F_TO_HUB_JOB15045
M16AM16A.QUE.ALL.6F_TO_HUB_JOB8024
M16BM16B.QUE.ALL.10F_TO_HUB_JOB3010
M16M16.QUE.SFAB.SENDQUEUETOTAL2010

10분 임계 = max(10, int(30분 임계 × 0.3)) 으로 자동 계산.

2-1. RB (30분 누적) — +10점

조건: 현재 값 - 30분 전 값 ≥ TH_RB_30[area]
사용자 예시: M14TOM16.MESCURRENTQCNT 가 30분 전 50 → 현재 155 → 차이 +105 ≥ 100 → RB trigger ON → M16HUB 영역 점수 +10

2-2. RB_fast (10분 폭주) — +5점

조건: 현재 값 - 10분 전 값 ≥ TH_RB_10[area]
예: 10분 전 80 → 현재 115 → 차이 +35 ≥ 30 → RB_fast ON → +5
30분 vs 10분 — 다른 사건 유형
• RB (30분): 점진적 데드락 — 큐가 천천히 누적
• RB_fast (10분): 폭주 — Bridge OHT Error 같은 급변

3. R-C (반전 흐름)

총량은 변하지 않거나 줄어들었는데 흐름 모양이 깨진 사건

3-1. M16HUB 리프터 반전 — +8점

대상: 10대 (LIFTER_IDS)

['6ABL6011', '6ABL6012', '6ABL6021', '6ABL6022',
 '6ABL6031', '6ABL6032',
 '6ABL0111', '6ABL0112', '6ABL0121', '6ABL0122']
조건:
  (a) rc_trend < 0 — 20분 전 대비 10대 합계 감소
  (b) rev_count ≥ TH_RC_REVERSE = 2 — 개별 증가 리프터 수 ≥ 2대
→ 둘 다 만족하면 RC trigger ON → +8

의미: 일부 리프터는 큐가 빠지고, 다른 리프터는 큐가 쌓이고 있음. 한쪽으로 쏠림.

3-2. M14 CNV 남북 편향 — +8점

북측 (4AFC3301) 과 남측 (4AFC3201) 간 편향.

조건: max(north, south) / (north + south) ≥ 0.70
예: 북 280, 남 70 → 280 / 350 = 0.80 ≥ 0.70 → RC trigger ON → M14 영역 +8

4. R-D (Fab 적체)

저장 공간 / 처리 용량이 한계에 다다른 사건

4-1. M16HUB Fab 적체 — +7점

조건 (OR — 둘 중 하나라도):
  (a) M16HUB.STRATE.ALL.FABSTORAGERATIO ≥ 25.0 (TH_RD_FABSTORAGE)
  (b) M16HUB.STRATE.STB.3F_STORAGE_UTIL ≥ 99.0 (TH_RD_HUB_STB_UTIL)
→ RD trigger ON → M16HUB 영역 +7

4-2. 다른 영역 OHT 가동률 — +7점

대상: M14 / M14B / M16A / M16B

조건: *.QUE.OHT.OHTUTIL ≥ 95.0 (TH_RD_OHT_UTIL)
예: M14.QUE.OHT.OHTUTIL = 97.3 → RD ON → M14 영역 +7
왜 임계 25 / 99 / 95?
• 25% — M16HUB 평소 fab 저장 분포의 p95
• 99% — STB 가 사실상 꽉 찬 시점 (마지막 1개 공간만)
• 95% — OHT 신규 작업 받을 여유 사라지는 임계

5. 영역 점수 (0~50)

각 영역마다 발동된 룰의 점수를 합산. 최대 50 클리핑.

시그널 트리거 (flag) 점수 비고
RA ra_trig +10 R-A 절대 임계
RA_sus ra_sustained +5 R-A 지속 임계
RB rb_trig +10 R-B 30분 누적
RB_fast rb_fast +5 R-B 10분 폭주
RC rc_trig +8 R-C 반전
RD rd_trig +7 R-D 적체
SLA sla_trig +5 전송 지연 (보조)
SORT sorter_trig +3 소터 대기 (보조)
MAXCAPA maxcapa_changed_n > 0 +10 × n 운영자 변경 (n = 변경 컬럼 수)
영역 점수 공식
area_score = min(50, RA_pts + RA_sus_pts + RB_pts + RB_fast_pts + RC_pts + RD_pts + SLA_pts + SORT_pts + MAXCAPA_pts)

6. 통합 점수 unified_risk_score (0~500)

최종 공식 (5 항 합산) 
unified_risk_score = min(500,
    layer1_total                    # 8 영역 점수 합산
  + flow_score                      # 10 노드 흐름 점수
  + sla_score                       # 영역별 SLA 보너스
  + sorter_score                    # 영역별 Sorter 보너스
  + mc_score                        # MAXCAPA 변경 보너스
)

6-1. layer1_total — 영역 점수 합산

layer1_total = Σ area_score (8 영역: M16HUB + M14 + M14B + M16A + M16B + M16 + M16_PKT + M16_WT)
최대 = 50 × 8 = 400

6-2. flow_score — 흐름 노드 점수

10 노드: M14_CNV_TO_HUB, M14_TO_HUB_JOB, M14B_7F_TO_HUB, M14B_LFT_4ABLD_SUM, M14B_LFT_4ABLD_TO_HUB_SUM, M16A_6F_TO_HUB, M16A_2F_TO_HUB, M16B_10F_TO_HUB, HUB_OHT_QCNT, M14_TO_M16

등급ratio 조건점수
주의≥ 1.5 (TH_FLOW_X1_5)+5
위험≥ 2.0 (TH_FLOW_X2_0)+15
심각≥ 3.0 (TH_FLOW_X3_0)+30

ratio = 현재 값 / baseline (10분 전 또는 평소 분포 기준). 10 노드 각각 평가하여 합산.

6-3. sla_score / sorter_score — 영역별 보너스

sla_score = 5 × (sla_trig 영역 수)
sorter_score = 3 × (sorter_trig 영역 수)

6-4. mc_score — MAXCAPA 변경

mc_score = 10 × (모든 영역의 maxcapa_changed_n 합)

8. Stage 판정 (S1 / S2 / S3)

any_ra          = 어느 영역이든 ra_trig OR ra_sustained
any_rb          = 어느 영역이든 rb_trig OR rb_fast
any_rd_or_sla   = 어느 영역이든 rd_trig OR sla_trig
any_rc          = M16HUB 의 rc_trig
any_flow_severe = 어느 흐름 노드든 '위험' 또는 '심각'

S1 = any_ra OR any_ra_sustained                             ← 조기경보
S2 = any_rb                                                 ← 주의보
S3 = any_ra AND (any_rd_or_sla OR any_rc) AND (any_rb OR any_flow_severe)  ← ★ 확정
Stage라벨의미
0이벤트없음모든 룰 정상
11단계 조기경보R-A 단독 — 시간 지연 신호만
22단계 주의보R-B 단독 — 큐 누적만
33단계 ⭐확정RA + (RD/SLA 또는 RC) + (RB 또는 flow severe) 3중 일치
왜 S3 가 3중 조건? 단일 룰 위양성 방지. 시간/공간/누적 3가지 시각이 동시 일치할 때만 사건 확정.
