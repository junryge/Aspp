[예시 1: CONVEYOR_Normal_CNV_Only.csv - 정상 이송]
--------------------------------------------------------------------------------
파일 특성:
  - 레코드: 10건
  - 메시지: INTERRAIL-* (컨베이어 전용)
  - 시간: 17:33:07 ~ 17:41:14 (약 8분)
  - 캐리어: 4PDML395
  - 경로: 4AFC3301A_IN58 → 4AFC3301A_OUT89

사용자 질문: "이 파일 분석해줘"

응답 예시:
```
이 파일은 M14A 컨베이어의 이송 로그예요.

캐리어 4PDML395가 컨베이어를 타고 이동한 기록인데요:

📍 경로: IN58 포트 → OUT89 포트
⏱️ 소요시간: 약 8분 6초
✅ 결과: 정상 완료

이벤트 순서:
1. 17:33:07 - IN58에서 바코드 스캔됨
2. 17:33:07 - 포트에 안착 확인 (ZONE_32606)
3. 17:33:10 - 이송 명령 발행 → MCS 승인
4. 17:33:10 - 이송 시작
5. 17:40:41 - OUT89에 도착, 바코드 재스캔
6. 17:40:41 - 이송 완료 (ZONE_10126)
7. 17:41:14 - 캐리어 반출

모든 이벤트가 WELL(정상) 상태이고, 시퀀스도 정상이에요. 
컨베이어 구간에서 문제없이 이송 완료된 케이스입니다.
```

[예시 2: CONVEYOR_Normal_with_FABJOB.csv - FABJOB 포함]
--------------------------------------------------------------------------------
파일 특성:
  - 레코드: 100건
  - 메시지: INTERRAIL-* + VM-* + UI-* 혼합
  - 시간: 17:33:05 ~ 17:41:15 (약 8분)
  - FAB 이동: M14 3F → M16 6F

사용자 질문: "FAB 이동 상태 알려줘"

응답 예시:
```
이 파일은 컨베이어 이벤트 + FAB간 이송 JOB 정보가 함께 있어요.

📦 캐리어: 4PDML395
📍 FAB 이동: M14A(3층) → M16A(6층)

컨베이어 구간:
- 입구: 4AFC3301A_IN58 (17:33:07)
- 출구: 4AFC3301A_OUT89 (17:40:41)
- 소요: 약 7분 34초

FAB JOB 상태:
- VM-FABTRANSPORTJOBCREATED: JOB 생성됨
- VM-FABTRANSPORTLOCATIONCHANGED: 위치 변경 보고 중
- 현재 컨베이어 구간 통과 중

✅ 컨베이어 구간은 정상 완료
⏳ 이후 Lifter → M16A OHT 구간 진행 예정
```

[예시 3: 바코드 스캔 에러]
--------------------------------------------------------------------------------
파일 특성:
  - IDREADSTATUS가 0이 아닌 경우
  - CARRIERIDREAD에서 에러

사용자 질문: "바코드 에러 원인?"

응답 예시:
```
⚠️ 바코드 스캔 에러가 발생했어요.

에러 내용:
- 위치: 4AFC3301A_IN58
- IDREADSTATUS: 2
- OLDCARRIERID: (비어있음)

IDREADSTATUS 코드:
- 0: 정상 스캔
- 1: 읽기 실패
- 2: ID 불일치
- 3: 중복 ID

🔍 확인 필요:
1. 캐리어 바코드 라벨 상태
2. 스캐너 청소 상태
3. 캐리어 위치 정렬 상태
```

[예시 4: 이송 지연]
--------------------------------------------------------------------------------
파일 특성:
  - IN → OUT 소요시간이 10분 이상

사용자 질문: "왜 이렇게 오래 걸렸어?"

응답 예시:
```
⚠️ 컨베이어 이송이 평소보다 오래 걸렸어요.

📍 경로: IN58 → OUT89
⏱️ 소요시간: 12분 30초 (정상: 7~8분)

지연 구간 분석:
- TRANSFERINITIATED → TRANSFERRING: 정상 (0.2초)
- TRANSFERRING → CARRIERIDREAD(출구): 11분 (지연!)

가능한 원인:
1. 컨베이어 정체 (앞에 다른 캐리어)
2. 컨베이어 속도 저하
3. 중간 구간 일시 정지

💡 해당 시간대 다른 캐리어 로그도 확인해보세요.
```
