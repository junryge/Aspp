[예시 1: OHT_Normal_OHT_Only.csv - 정상 이송]
--------------------------------------------------------------------------------
파일 특성:
  - 레코드: 14건
  - 메시지: RAIL-* (OHT 전용)
  - 시간: 17:31:10 ~ 17:33:04 (약 2분)
  - 캐리어: 4PDML395
  - 경로: 4ANZ53-411 → 4AFC3301A_IN58

사용자 질문: "이 파일 분석해줘"

응답 예시:
```
이 파일은 M14A OHT의 이송 로그예요.

캐리어 4PDML395가 OHT(V00922)를 타고 이동한 기록인데요:

📍 경로: Stocker(4ANZ53-411) → Conveyor 입구(4AFC3301A_IN58)
⏱️ 소요시간: 약 1분 54초
✅ 결과: 정상 완료 (RESULTCODE=0)

이벤트 순서:
1. 17:31:10 - 이송 명령 발행 (CARRIERTRANSFER)
2. 17:31:10 - MCS 승인 (HCACK=4)
3. 17:31:13 - V00922 차량 할당
4. 17:31:13 - 이송 시작
5. 17:32:51 - 하차 시작
6. 17:33:04 - 이송 완료, 차량 해제

모든 이벤트가 WELL(정상) 상태이고, 시퀀스도 정상이에요.
OHT 구간에서 문제없이 이송 완료된 케이스입니다.
```

[예시 2: OHT_TransferUpdate - 목적지 변경]
--------------------------------------------------------------------------------
파일 특성:
  - 메시지에 RAIL-TRANSFERUPDATE 포함
  - 목적지가 중간에 변경됨

사용자 질문: "왜 목적지가 바뀌었어?"

응답 예시:
```
이 로그에서 TRANSFERUPDATE 이벤트가 발생했어요.

📍 원래 목적지: 4AFC3301A_IN58
📍 변경된 목적지: 4AFC3301A_IN60

변경 사유는 로그에서 확인되지 않지만, 일반적으로:
- 원래 포트가 사용 중
- 우선순위 높은 캐리어가 먼저 배정됨
- 포트 장애로 인한 대체 경로 선택

⚠️ 이 경우 소요시간이 조금 더 걸릴 수 있어요.
변경 후에도 정상 완료되었으니 문제는 없습니다.
```

[예시 3: OHT_StageCommand - 스테이지 명령]
--------------------------------------------------------------------------------
파일 특성:
  - RAIL-STAGECOMMAND 메시지 포함
  - 특정 스테이지로 명령 발행

사용자 질문: "스테이지 명령이 뭐야?"

응답 예시:
```
STAGECOMMAND는 OHT에게 특정 위치(스테이지)로 이동하라는 명령이에요.

이 로그에서는:
📍 스테이지: STAGE_01
🚗 차량: V00922

일반 이송(CARRIERTRANSFER)과 다른 점:
- 캐리어 없이 차량만 이동
- 대기 위치로 복귀하거나
- 다음 작업을 위해 미리 이동

정상적인 운영 명령이에요.
```

[예시 4: 에러 케이스]
--------------------------------------------------------------------------------
파일 특성:
  - LEVEL에 ERROR 또는 WARN 포함
  - RESULTCODE가 0, 4가 아닌 경우

사용자 질문: "에러 원인 분석해줘"

응답 예시:
```
⚠️ 이 로그에서 에러가 발견되었어요.

에러 내용:
- 메시지: RAIL-CARRIERTRANSFERREPLY
- RESULTCODE: 6
- LEVEL: ERROR

RESULTCODE 6은 일반적으로:
- 목적지 포트가 사용 불가
- 캐리어가 이미 다른 명령 수행 중
- 차량 할당 실패

🔍 확인 필요 사항:
1. 해당 시간대 목적지 포트 상태
2. 캐리어의 이전 명령 완료 여부
3. OHT 차량 가용 상태
```
