# 📦 AI 물류 예측 시스템 - 13개 핵심 지표 설명

**"3개 공장(M14A, M14B, M10A)이 서로 물건을 주고받는 상황을 실시간으로 보는 지표들입니다"**

---

## 🏭 1. 공장 간 물류량 (7개) - "트럭 대수"

### 📤 보내는 양 (4개)

| 번호 | 컬럼명 | 설명 |
|------|--------|------|
| 1 | M14AM14B | M14A가 M14B로 보낸 트럭 대수 |
| 2 | M14AM14BSUM | M14A→M14B 오늘 총 누적 트럭 |
| 3 | M14AM10A | M14A가 M10A로 보낸 트럭 대수 |
| 4 | M14AM16SUM | M14A→M16 오늘 총 누적 트럭 |

**쉽게 말하면:**
```
"우리 공장(M14A)에서 다른 공장으로 몇 대 보냈나?"
- 트럭 많이 보냄 = 바쁨
- 특히 M14B로 500대 이상 = 위험 신호! 🚨
```

### 📥 받는 양 (3개)

| 번호 | 컬럼명 | 설명 |
|------|--------|------|
| 5 | M14BM14A | M14B가 M14A로 보낸 트럭 (역방향) |
| 6 | M10AM14A | M10A가 M14A로 보낸 트럭 (역방향) |
| 7 | M16M14A | M16이 M14A로 보낸 트럭 (역방향) |

**쉽게 말하면:**
```
"다른 공장에서 우리한테 몇 대 들어왔나?"
- 들어오는 양 많음 = 우리가 처리할 게 많아짐
```

## 🚛 3. 대기/처리 상황 (4개) - "교통 상황"

### 📋 작업 대기 (2개)

| 번호 | 컬럼명 | 설명 |
|------|--------|------|
| 9 | CURRENTQCREATED | 오늘 발생한 작업 요청 수 |
| 10 | CURRENTQCOMPLETED | 오늘 완료한 작업 수 |

**식당으로 비유하면:**

```
CREATED   = 주문 들어온 수 (1000건)
COMPLETED = 요리 완성된 수 (950건)
차이 50건 = 지금 주방에서 만들고 있는 중
```

### ⚖️ queue_gap (가장 중요! 🏆)

```python
queue_gap = CREATED - COMPLETED
```

**의미:**
- 지금 밀려있는 작업량
- 대기 중인 일의 양

| gap 크기 | 상태 | 설명 |
|----------|------|------|
| 50건 | ✅ 정상 | 주방 여유 있음 |
| 150건 | ⚠️ 주의 | 약간 밀리기 시작 |
| 300건 | 🔥 위험 | 주방 터짐! |
| 500건+ | 🚨 폭발 | 감당 불가! |

## 💡 가장 중요한 3가지 핵심 지표

### 🥇 1위: queue_gap (작업 대기량)

```
= 주문 들어온 수 - 완료한 수
= 지금 밀려있는 일
```

| gap | 상태 |
|-----|------|
| 50 | ✅ 정상 |
| 300 | 🔥 위험! |
| 500+ | 🚨 폭발 직전! |

**왜 중요?**
- 일이 얼마나 밀렸는지 직접적으로 보여줌
- 1700+ 예측 정확도 가장 높음
- 상관계수 4.2배!

---

### 🥈 2위: M14AM14B (핵심 물류)

```
= M14A→M14B 트럭 대수
```

| 대수 | 상태 |
|------|------|
| 300대 미만 | ✅ 정상 |
| 450대 이상 | ⚠️ 주의 |
| 500대 이상 | 🚨 위험! |

**왜 중요?**
- M14B는 가장 바쁜 라인
- 500대 넘으면 70% 확률로 1700+ 발생
- 황금 패턴의 핵심 지표

---

### 🥉 3위: TRANSPORT 초과 (배송 지연)

```
= 4분 넘게 걸린 배송 건수
```

| 건수 | 상태 |
|------|------|
| 30건 미만 | ✅ 정상 |
| 150건 이상 | ⚠️ 막힘 |
| 200건 이상 | 🚨 대란! |

**왜 중요?**
- 운송 병목 현상 직접 표시
- 지연이 많으면 전체 시스템 마비
- 상관계수 1.29배
