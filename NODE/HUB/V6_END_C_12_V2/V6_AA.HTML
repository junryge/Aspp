XGBoost 기반 반도체 제조 라인 HUB 급증 조기 예측 시스템

Version 6.0 - Momentum Enhanced

📋 시스템 개요

🎯 핵심 목적

반도체 제조 라인의 HUB 시스템에서 급격한 큐 증가(급증)를 사전에 감지하여, 운영 중단이나 병목 현상을 예방하는 AI 기반 예측 시스템입니다.

⏱️
예측 방식
최근 30분간의 데이터를 분석하여 다음 10분 내 최대값을 예측합니다.
🎯
감지 목표
정상 범위(<300)에서 갑자기 위험 수준(≥300)으로 급증하는 케이스를 조기 탐지합니다.
💡 왜 이 시스템이 중요한가?
제조 라인에서 HUB 큐가 300을 초과하면 전체 라인의 흐름이 멈추거나 심각한 병목이 발생할 수 있습니다. 이 시스템은 급증이 발생하기 전에 미리 경고하여 운영팀이 선제적으로 대응할 수 있게 합니다.

🆕 V6의 혁신 - Momentum Features

기존 모델은 단순히 과거 평균이나 최대값만 보았다면, V6는 "급증 직전의 가속도 패턴"을 감지합니다.

  • ✓ 가속도(Acceleration): 최근 5분과 그 이전 5분의 평균 차이로 상승 속도 측정
  • ✓ 상승 추세(Is Rising): 현재 값이 5분 전보다 높은지 확인
  • ✓ 급등 패턴(Rapid Rise): 5분간 10 이상 급증했는지 감지
  • ✓ 단기 평균(Last 10min Mean): 최근 10분 평균으로 단기 추세 파악

📊 1단계: 통계 분석
30분 데이터의 평균, 표준편차, 최대/최소값, 기울기 등 기본 통계 계산
🚀 2단계: Momentum 분석 (신규!)
가속도, 상승 추세, 급등 패턴, 단기 평균을 계산하여 급증 직전 신호 포착
🎯 3단계: 위험 요인 분석
HUB 병목, 스토리지 과부하, CMD 부족 등 복합 위험 조건 체크
🔥 4단계: 급증 임박 신호
값 > 280 + 가속도 > 0.5 + HUB 병목 조건이 동시에 만족되면 "급증 임박" 신호 발생

🔥 급증 위험도 점수 계산식

위험도 = (HUB병목 × 3) + (스토리지과부하 × 2) + (CMD부족 × 1) + (FS부족 × 1)

  • ✓ 위험도 ≥ 5: CRITICAL (즉시 대응 필요)
  • ✓ 위험도 ≥ 3: HIGH (주의 필요)
  • ✓ 위험도 ≥ 1: MEDIUM (모니터링 강화)

❓ 자주 묻는 질문 (FAQ)

Q1. 왜 10분 후를 예측하나요?

운영팀이 실제로 대응 조치를 취하기 위해서는 최소 5-10분의 시간이 필요합니다. 더 짧으면 대응 시간이 부족하고, 더 길면 예측 정확도가 떨어집니다. 10분은 정확도와 실용성의 최적 균형점입니다.

Q2. 왜 30분 데이터가 필요한가요?

급증은 갑자기 발생하는 것처럼 보이지만, 실제로는 30분 정도의 전조 패턴이 있습니다. 더 짧은 시간(예: 10분)만 보면 패턴 파악이 어렵고, 더 긴 시간(예: 60분)은 불필요한 과거 데이터까지 포함하게 됩니다. 30분이 급증 패턴을 포착하는 최적 윈도우 크기입니다.

Q3. 60-70% 감지율이면 충분한가요?

예, 충분합니다. 이유:

  • ✓ 급증은 매우 예측하기 어려운 현상입니다 (갑작스럽고 비선형적)
  • ✓ 기존에는 0% 감지(사후 대응만 가능)였습니다
  • ✓ 60-70%만 예방해도 라인 정지 빈도가 획기적으로 감소합니다
  • ✓ 나머지 30-40%도 지속적인 개선으로 감지율 향상 중입니다

100% 감지는 이상적이지만, 60-70%도 실무적으로 매우 높은 가치입니다!

Q4. 오탐(False Alarm)은 얼마나 발생하나요?

임계값 280은 낮은 오탐률을 유지하도록 최적화되었습니다. 일반적으로 실제 급증이 아닌데 경보가 발생하는 경우는 10-15% 수준입니다. 이는 실무적으로 충분히 수용 가능한 수준이며, 보수적으로 운영하는 것이 라인 정지보다 훨씬 낫습니다.

Q5. 모델은 얼마나 자주 재학습해야 하나요?

권장 재학습 주기:

  • ✓ 정기: 월 1회 (새로운 데이터 반영)
  • ✓ 임시: 제조 라인 구성 변경 시
  • ✓ 긴급: 감지율 급락 시

다만, 현재 모델도 충분히 강건하므로 급한 재학습은 필요하지 않습니다.

Q6. 다른 제조 라인에도 적용 가능한가요?

예, 가능합니다! 핵심 로직은 동일하며, 다음만 조정하면 됩니다:

  • ✓ 데이터 컬럼명 매핑
  • ✓ 임계값 조정 (라인별 특성 반영)
  • ✓ 해당 라인 데이터로 재학습

시스템 구조는 범용적으로 설계되어 확장이 용이합니다!
